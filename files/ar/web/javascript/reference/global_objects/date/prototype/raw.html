{{JSRef("Global_Objects", "Date")}}

Summary

The Date.prototype property represents the prototype for the {{jsxref("Global_Objects/Date", "Date")}} constructor.

{{js_property_attributes(0,0,1)}}

Description

JavaScript {{jsxref("Global_Objects/Date", "Date")}} instances inherit from {{jsxref("Global_Objects/Date/prototype", "Date.prototype")}}. You can modify the constructor's prototype object to affect properties and methods inherited by JavaScript Date instances.

For compatibility with millennium calculations (in other words, to take into account the year 2000), you should always specify the year in full; for example, use 1998, not 98. To assist you in specifying the complete year, JavaScript includes the methods {{jsxref("Global_Objects/Date/getFullYear", "getFullYear()")}}, {{jsxref("Global_Objects/Date/setFullYear", "setFullYear()")}}, {{jsxref("Global_Objects/Date/getUTCFullYear", "getUTCFullYear()")}} and {{jsxref("Global_Objects/Date/setUTCFullYear", "setUTCFullYear()")}}.

Properties

Date.prototype.constructor
Returns the function that created an instance. This is the {{jsxref("Global_Objects/Date", "Date")}} Date constructor by default.
