Das RTCDataChannel Interface repräsentiert einen bidirektionalen Datenkanal zwischen zwei Endpunkten einer direkten Verbindungen.

Ein Objekt dieses Types kann mit {{domxref("RTCDataChannel.createDataChannel()")}} erstellt, oder als Rückgabewert des Events {{event("datachannel")}} als Typ {{domxref("RTCDataChannelEvent")}} bei einer bestehenden {{domxref("RTCPeerConnection")}} zurückgegeben werden.

In Gecko ist die API über DataChannel anstelle des Standard RTCDataChannel namens erreichbar.

Properties

{{domxref("RTCDataChannel.label")}} {{readOnlyInline}}
Gibt ein {{domxref("DOMString")}} zurück. Dieser beinhaltet den Namen, welcher den Datenkanal beschreibt. Es gibt keine Einschränkung der Einzigartigkeit.
{{domxref("RTCDataChannel.ordered")}} {{readOnlyInline}}
Gibt ein {{domxref("Boolean")}} zurück, der aussagt ob das Empfangen der Nachricht garantiert ist, oder nicht
{{domxref("RTCDataChannel.protocol")}} {{readOnlyInline}}
Returns a {{Domxref("DOMString")}} containing the name of the subprotocol in use. If none, it returns "".
{{domxref("RTCDataChannel.id")}} {{readOnlyInline}}
Returns an unsigned short being a unique id for the channel. It is set at the creation of the {{domxref("RTCDataChannel")}} object.
{{domxref("RTCDataChannel.readyState")}} {{readOnlyInline}}
Returns an enum of the type RTCDataChannelState representing the state of the underlying data connection. It can be one of the following values:
  • "connecting" is the state indicating that the underlying connection is not yet set up and active. This is the initial state of a data channel created with {{domxref("RTCPeerConnection.createDataChannel()")}}.
  • "open" is the state indicating that the underlying connection is up and running. This is the initial state of a data channel dispatched in a {{domxref("RTCDataChannelEvent")}}.
  • "closing" is the state indicating that the underlying connection is in the process of shutting down. No new sending task is accepting but the cached messages are in the process of being sent, or received.
  • "closed" is the state indicating that the underlying connection has been shut down (or couldn't be established).
{{domxref("RTCDataChannel.bufferedAmount")}} {{readOnlyInline}}
Returns an unsigned long containing the amount of bytes that have been queued for sending: that is the amount of data requested to be transmitted via {{domxref("RTCDataChannel.send()")}} that has not been sent yet. Note that if the channel is closed, the buffering continues.
{{domxref("RTCDataChannel.binaryType")}}
Is a {{domxref("DOMString")}} indicating the type of binary data transmitted by the connection. This should be either "blob" if {{domxref("Blob")}} objects are being used or "arraybuffer" if {{domxref("ArrayBuffer")}} objects are being used. Initially it is set to "blob".
{{domxref("RTCDataChannel.maxPacketLifeType")}} {{readOnlyInline}}
Is an unsigned short indicating the length in milliseconds of the window in when messaging happens in unreliable mode.
{{domxref("RTCDataChannel.maxRetransmits")}} {{readOnlyInline}}
Is an unsigned short indicating the maximum amount of retransmissions that can happen when messaging happens in unreliable mode.
{{domxref("RTCDataChannel.negotiated")}} {{readOnlyInline}}
Is a {{domxref("Boolean")}} indicating if the channel has been negotiated by the application, or not.
{{domxref("DataChannel.reliable")}} {{non-standard_inline}} {{readOnlyInline}}
Is a {{domxref("Boolean")}} indicating if the connection can send message in unreliable mode.
{{domxref("DataChannel.stream")}} {{non-standard_inline}} {{readOnlyInline}}
Is an obsolete synonym for {{domxref("RTCDataChannel.id")}}.

Event Handlers

{{domxref("RTCDataChannel.onopen")}}
Is the event handler called when the {{event("open")}} event is received. Such an event is sent when a the underlying data transport, that is the data connection, has been established.
{{domxref("RTCDataChannel.onmessage")}}
Is the event handler called when the {{event("message")}} event is received. Such an event is sent when a message is available on the data connection.
{{domxref("RTCDataChannel.onclose")}}
Is the event handler called when the {{event("close")}} event is received. Such an event is sent when the underlying data transport has been closed.
{{domxref("RTCDataChannel.onerror")}}
Is the event handler called when the {{event("error")}} event is received. Such an event is sent when an error has been encountered.

Methods

{{domxref("RTCDataChannel.close()")}}
Closes the channel. The closing is done in a non abrupt way. The {{domxref("RTCDataChannel.readyState", "state")}} of the channel is set to "closing", the messages not yet sent are sent, then the channel is closed.
{{domxref("RTCDataChannel.send()")}}
Sends the data in parameter over the channel. The data can be a {{domxref("DOMString")}}, a {{domxref("Blob")}}, an {{domxref("ArrayBuffer")}} or an {{domxref("ArrayBufferView")}}.

Example

var pc = new RTCPeerConnection();
var dc = pc.createDataChannel("my channel");

dc.onmessage = function (event) {
  console.log("received: " + event.data);
};

dc.onopen = function () {
  console.log("datachannel open");
};

dc.onclose = function (
  console.log("datachannel close");
};
