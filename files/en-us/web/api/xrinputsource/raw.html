The XRInputSource interface of the {{domxref('WebXR Device API','','',' ')}} returns information about the Web AR/VR control device being used. The control device is platform-specific and defines a primary action. A primary action is a trigger, touchpad, button, spoken command, or hand gesture that when performed produces {{domxref("XRSession.selectstart","selectstart")}}, {{domxref("XRSession.selectend","selectend")}}, and {{domxref("XRSession.select","select")}} events.

Properties

{{domxref('XRInputSource.gripSpace')}}{{readonlyInline}}{{experimental_inline}}
Returns an {{domxref("XRSpace")}} whose native origin tracks the pose that should be used to render virtual objects such that they appear to be held in the user's hand.
{{domxref('XRInputSource.handedness')}}{{readonlyInline}}{{experimental_inline}}
Describes which hand the XRInputSource is associates with, if any. This will be one of "left", "right", or "none".
{{domxref('XRInputSource.profiles')}}{{readonlyInline}}{{experimental_inline}}
Returns an array of input profile names indicating both the preferred visual representation and behavior of the input source.
{{domxref('XRInputSource.targetRayMode')}}{{readonlyInline}}{{experimental_inline}}

Returns the method used to produce the target ray. This will be one of "gaze", "tracked-pointer", or "screen".

{{domxref('XRInputSource.targetRaySpace')}}{{readonlyInline}}{{experimental_inline}}
Returns an {{domxref("XRSpace")}} with a native origin tracking the position and orientation of the preferred point ray as defined by {{domxref("XRInputSource.targetRayMode")}}.
