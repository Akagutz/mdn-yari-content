The Date.prototype property represents the prototype for the {{jsxref("Date")}} constructor.

{{js_property_attributes(0, 0, 1)}}

Description

JavaScript {{jsxref("Date")}} instances inherit from Date.prototype. You can modify the constructor's prototype object to affect properties and methods inherited by JavaScript {{jsxref("Date")}} instances.

For compatibility with millennium calculations (in other words, to take into account the year 2000), you should always specify the year in full; for example, use 1998, not 98. To assist you in specifying the complete year, JavaScript includes the methods {{jsxref("Date.prototype.getFullYear()", "getFullYear()")}}, {{jsxref("Date.prototype.setFullYear()", "setFullYear()")}}, {{jsxref("Date.prototype.getUTCFullYear()", "getUTCFullYear()")}} and {{jsxref("Date.prototype.setUTCFullYear()", "setUTCFullYear()")}}.

Starting with ECMAScript 6, the Date.prototype object is itself an ordinary object. It is not a {{jsxref("Date")}} instance.

Properties

Date.prototype.constructor
Returns the function that created an instance. This is the {{jsxref("Date")}} constructor by default.
