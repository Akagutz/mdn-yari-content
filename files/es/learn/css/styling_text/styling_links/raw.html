A la hora de dar estilo a los enlaces, es importante comprender cómo utilizar las pseudo-clases para diseñar los estados de un enlace de manera efectiva y cómo diseñar enlaces para su uso en funciones de interfaz variadas comunes, como menús y pestañas de navegación. Veremos todos estos temas en este artículo.

Prerrequisitos: Conocimientos básicos de informática, conceptos básicos de HTML (véase Introducción al HTML), CSS básico (véase Introducción al CSS), texto CSS y los fundamentos de la fuente.
Objetivo: Aprender a dar formato a los estados de un enlace y cómo usarlos de manera efectiva en características comunes de la interfaz de usuario como los menús de navegación.

Echemos un vistazo a algunos enlaces

Analizamos cómo se implementan los enlaces en su HTML de acuerdo con las mejores prácticas de Creación de hipervínculos. En este artículo desarrollaremos este conocimiento y te mostraremos las mejores prácticas para diseñar enlaces.

Estados de enlace

Lo primero que hay que entender es el concepto de estados de enlace (diferentes estados en los que pueden existir enlaces, que pueden diseñarse usando diferentes pseudo-clases):

  • Link (no visitado): El estado predeterminado en el que reside un enlace cuando no está en ningún otro estado. Se puede especificar usando la pseudo-clase {{cssxref(":link")}}.
  • Visited: Cuando ya se ha visitado (queda grabado en la historia del navegador), la utiliza la pseudo-clase {{cssxref(":visited")}} para darle otro formato.
  • Hover: Daremos formato al enlace para que aparezca diferente cuando se le pasa el cursor por encima con la pseudo-clase {{cssxref(":hover")}}.
  • Focus: El estado de un enlace cuando la atención se centra sobre él (por ejemplo, se ha pasado por encima usando la tecla Tab o mediante programación enfocada usando {{domxref("HTMLElement.focus()")}}) que juntamente con la pseudo-clase {{cssxref(":focus")}} le dan formato.
  • Active: Un enlace cuando se ha activado (por ejemplo, se ha hecho clic encima) al cual se le da formato con la pseudo-clase {{cssxref(":active")}}.

Estilos predeterminados

El siguiente ejemplo ilustra como se comportará un enlace por defecto ( el CSS simplemente amplía y centra el texto para que se destaque más).

<p><a href="#">A simple link</a></p>

p {
  font-size: 2rem;
  text-align: center;
}

Nota: Todos los enlaces en los ejemplos de esta página son enlaces falsos, encontrarás un símbolo # (almohadilla) en el lugar de la URL real. Esto se debe a que si incluyéramos los enlaces reales, los ejemplos dejarían de ser válidos en caso de que se hiciera clic encima o una acción similar (daría error o con una página cargada en el ejemplo incrustado que no se podía recuperar). # solo enlaza dentro de la página actual.

Notarás algunas cosas a medida que explores los estilos predeterminados:

  • Los enlaces están subrayados.
  • Los enlaces no visitados se ven de color azul.
  • Los enlaces visitados se ven de color morado.
  • Al pasar el cursor sobre un enlace, el puntero del ratón cambia a un pequeño icono de mano.
  • Los enlaces enfocados tienen un esquema alrededor de ellos: debe poder enfocarse en los enlaces de esta página con el teclado presionando la tecla de tabulación (en Mac, es posible que necesites habilitar la opción Acceso total al teclado: Todos los controles presionando Ctrl + F7 par aque funcione).
  • Los enlaces que se han activado se muestran de color rojo (intenta mantener presionado el cursor sobre el enlace mientras haces clic).

Curiosamente, estos estilos predeterminados son casi los mismos que se utilizaban al inicio de los navegadores en la década de 1990. Esto se debe a que los usuarios conocen y esperan que sea de esta forma. Si los enlaces empezaran a mostrarse de manera diferente, mucha gente se mostraría confusa. Esto no quiere decir que no puedas cambiar el formato de los enlaces, solo que no deberías alejarte mucho de la conducta esperada. Al menos deberías:

  • Subrayar los enlaces, pero no otros elementos. Si no quieres hacerlo, al menos destacarlos de alguna otra forma.
  • Hacer que reaccionen de alguna manera cuando se pasa el cursor por encima y de una manera ligeramente diferente cuando se activan.

Los estilos por defecto se pueden desactivar / cambiar usando las siguientes propiedades CSS:

  • {{cssxref("color")}} para el color del texto.
  • {{cssxref("cursor")}} para el estilo del cursor (no debes deshabilitar esta opción a menos que tengas una muy buena razón).
  • {{cssxref("outline")}} para el esquema de texto (un esquema es similar a una frontera, la única diferencia es que la frontera ocupa espacio en la caja y el esquema sólo se encuentra en la parte superior del fondo). El esquema es una ayuda de accesibilidad útil, así que piensa cuidadosamente antes de apagarlo; al menos deberías duplicar los estilos dados al estado de desplazamiento y de enfoque.

Nota: No estás limitado a las propiedades anteriores para diseñar tus enlaces, sino que puedes usar cualquier propiedad que desees. ¡Solo trata de no cometer locuras!

Dar formato a algunos enlaces

Ahora que hemos visto los estados predeterminados, veamos un conjunto típico de estilos de enlace.

Para empezar, escribiremos nuestros conjuntos de reglas vacías:

a {

}


a:link {

}

a:visited {

}

a:focus {

}

a:hover {

}

a:active {

}

Este orden es importante porque los estilos de enlace se construyen unos sobre otros. Por ejemplo, los estilos de la primera regla se aplicarán a todos los estados posteriores, a cuando se está activando un enlace y cuando se pasa por encima con el cursor. Si los escribes en el orden incorrecto, no funcionarán correctamente. Para recordar el orden, los programadores utilizan la nemotecnia LoVe Fears HAte.

Ahora vamos a añadir algo más de información para conseguir darles un formato correcto:

body {
  width: 300px;
  margin: 0 auto;
  font-size: 1.2rem;
  font-family: sans-serif;
}

p {
  line-height: 1.4;
}

a {
  outline: none;
  text-decoration: none;
  padding: 2px 1px 0;
}

a:link {
  color: #265301;
}

a:visited {
  color: #437A16;
}

a:focus {
  border-bottom: 1px solid;
  background: #BAE498;
}

a:hover {
  border-bottom: 1px solid;     
  background: #CDFEAA;
}

a:active {
  background: #265301;
  color: #CDFEAA;
}

También proporcionaremos algunos ejemplos de HTML para aplicar al CSS:

<p> Hay varios navegadores disponibles, tales como <a href="#">Mozilla
Firefox</a>, <a href="#">Google Chrome</a> y
<a href="#">Microsoft Edge</a>. </p>

Entonces, ¿qué hacemos? Sin duda tiene un aspecto diferente al estilo predeterminado, pero todavía ofrece una experiencia bastante familiar para que los usuarios sepan que pasa:

  • Las dos primeras reglas no son tan interesantes en esta situación.
  • La tercera regla usa el selector a para eliminar el subrayado de texto predeterminado y el esquema de enfoque (que de todos modos varía dependiendo del navegador), y agrega una pequeña cantidad de relleno a cada enlace (todo esto se aclarará más adelante).
  • A continuación, utilizamos los selectores a:link y a:visited para establecer un par de variaciones de color en los enlaces no visitados y visitados, por lo que son distintos.
  • Las siguientes dos reglas usan a:focus y a:hover para establecer que los enlaces tengan diferentes colores de fondo, además de un subrayado para que el enlace destaque aún más. Dos puntos a destacar son:
    • El subrayado se ha creado usando {{cssxref("border-bottom")}} y no {{cssxref("text-decoration")}} y es que algunas personas lo prefieren porque el primero tiene mejores opciones en cuanto a estilo, y se dibuja un poco más abajo, por lo que no atraviesa los descendientes de la palabra subrayada.
    • El valor {{cssxref ("border-bottom")}} se ha establecido como 1px solid sin ningún color especificado. Hacer esto hace que el borde adopte el mismo color que el texto del elemento, lo cual es útil en casos donde el texto es de un color diferente en cada caso.
  • Finalmente, a:active se usa para dar a los enlaces un esquema de color invertido mientras se activan, ¡para dejar claro que está sucediendo algo importante!

Incluir iconos de enlaces

Una práctica común es incluir iconos en los enlaces para proporcionar más de un indicador en cuanto a qué tipo de contenido redirige. Echemos un vistazo a un ejemplo muy simple que añade un icono a enlaces externos (enlaces que conducen a otros sitios). Tal ícono generalmente se parece a una pequeña flecha que señala un cuadro; para este ejemplo, usaremos este gran ejemplo de icons8.com.

Echemos un vistazo a algunos elementos HTML y CSS que nos darán el efecto que queremos. En primer lugar, algunos elementos HTML simples para dar estilo:

<p> Para obtener más información sobre el tiempo, visite nuestra <a href="http://#">página del tiempo </a>,
mire la <a href="http://#"> Wikipedia </a>, o cheque
<a href="http://#">Extreme Science</a>.</p>

A continuación, el CSS:

body {
  width: 300px;
  margin: 0 auto;
  font-family: sans-serif;
}

p {
  line-height: 1.4;
}

a {
  outline: none;
  text-decoration: none;
  padding: 2px 1px 0;
}

a:link {
  color: blue;
}

a:visited {
  color: purple;
}

a:focus, a:hover {
  border-bottom: 1px solid;
}

a:active {
  color: red;
}

a[href*="http"] {
  background: url('https://mdn.mozillademos.org/files/12982/external-link-52.png') no-repeat 100% 0;
  background-size: 16px 16px;
  padding-right: 19px;
}

Entonces, ¿qué está pasando? Omitiremos la mayor parte del CSS, ya que es la misma información que has visto antes. Sin embargo, la última regla es interesante: insertamos una imagen de fondo personalizada en enlaces externos de una manera similar a cómo manejamos viñetas personalizadas en los elementos de la lista en el último artículo. Esta vez, sin embargo, utilizamos la taquigrafía {{cssxref("background")}} en lugar de las propiedades individuales. Establecemos la ruta a la imagen que queremos insertar, especificamos que no se repita para que solo se inserte una copia, y luego especificamos la posición como el 100% del camino a la derecha del contenido del texto y 0 píxeles desde la parte superior .

También usamos {{cssxref("background-size")}} para especificar el tamaño de la imagen de fondo (es útil tener un ícono más grande y luego cambiar su tamaño de esta manera según sea necesario para fines de diseño web receptivo) Sin embargo, esto solo funciona con IE 9 y versiones posteriores, por lo que si trabaja con navegadores antiguos, solo tendrá que cambiar el tamaño de la imagen e insertarla como está.

Finalmente, configuramos algunos {{cssxref ("padding-right")}} en los enlaces para crear espacio para que aparezca la imagen de fondo, de modo que no se superponga con el texto.

Un último apunte: ¿cómo seleccionamos solo los enlaces externos? Bueno, si escribes los enlaces HTML correctamente, solo debe usar URL absolutas para enlaces externos. Resulta más eficiente usar enlaces relativos para vincular a otras partes del mismo sitio web. El texto "http" por lo tanto, sólo debe aparecer en los enlaces externos, y podemos seleccionar esto con un selector de atributos: a[href*="http"] selecciona los elementos {{HTMLElement ( "a")}}, pero sólo si tienen un atributo {{htmlattrxref("href","a")}} con un valor que contiene "http" en algún lugar dentro de él.

Eso es todo. ¡Intenta volver a visitar la sección de aprendizaje activo anterior y prueba esta nueva técnica!

Nota: No te preocupes si aún no estás familiarizado con los fondos y el diseño web receptivo, lo explicaremos en otros artículos.

El estilo de enlaces como botones

Las herramientas que has explorado hasta ahora en este artículo también se pueden usar de otras maneras. Por ejemplo, los estados como hover se pueden usar para diseñar muchos elementos diferentes, no solo enlaces. Es posible que desees diseñar el estado de desplazamiento de los párrafos, elementos de la lista u otras cosas.

Además, los enlaces tienen un estilo bastante común para verse y comportarse como botones en ciertas circunstancias: un menú de navegación del sitio web generalmente marcado como una lista que contiene enlaces, y esto puede diseñarse fácilmente para parecerse a un conjunto de botones de control o pestañas que proporcionan al usuario acceso a otras partes del sitio. Vamos a explorar cómo lo hacen.

En primer lugar, algo de HTML:

<ul>
  <Li> <a href="#"> Inicio </a> </ li> <li> <a href="#"> Pizza </a> </ li> <li> <a href = "#" > Música </a> </ li> <li> <a href="#"> Wombats </a> </ li> <li> <a href="#"> Finlandia </a> </ li>
</ul>

Y ahora nuestro CSS:

body,html {
  margin: 0;
  font-family: sans-serif;
}

ul {
  padding: 0;
  width: 100%;
}

li {
  display: inline;
}

a {
  outline: none;
  text-decoration: none;
  display: inline-block;
  width: 19.5%;
  margin-right: 0.625%;
  text-align: center;
  line-height: 3;
  color: black;
}

li:last-child a {
  margin-right: 0;
}

a:link, a:visited, a:focus {
  background: yellow;
}

a:hover {     
  background: orange;
}

a:active {
  background: red;
  color: white;
}

Vamos a explicar qué ha pasado, centrándonos en las partes más interesantes:

  • Nuestra segunda regla elimina el {{cssxref ("padding")}} predeterminado del elemento {{htmlelement ("ul")}} y establece su ancho para abarcar el 100% del contenedor externo (el {{htmlelement ("body ")}}, en este caso).
  • Los elementos {{htmlelement ("li")}} normalmente están bloqueados por defecto (vea los tipos de cuadros CSS para una actualización), lo que significa que permenecerán en sus propias líneas. En este caso, estamos creando una lista horizontal de enlaces, por lo que en la tercera regla establecemos la propiedad {{cssxref ("display")}}, lo que hace que los elementos de la lista se ubiquen en la misma línea entre sí: ahora se comportan como elementos en línea.
  • La cuarta regla (que da formato al elemento {{htmlelement("a")}} ) es el más complicado. Vamos a verlo paso a paso:
    • Como en los ejemplos anteriores, comenzamos desactivando los elementos {{cssxref ("text-decoration")}} y {{cssxref ("outline")}} predeterminados; ya que no queremos que estropeen nuestro aspecto.
    • A continuación, configuramos {{cssxref ("display")}} para inline-block: los elementos {{htmlelement ("a")}} se muestran en línea de forma predeterminada y, aunque no queremos que se derramen en sus propias líneas como lo lograría un valor de bloque, queremos poder dimensionarlos: inline-block nos permite hacerlo.
    • ¡Ahora a por el tamaño! Queremos llenar todo el ancho de {{htmlelement ("ul")}} pero dejar un pequeño margen entre cada botón (pero no un espacio en el borde derecho), y tenemos 5 botones para hacer que sean del mismo tamaño. Para hacer esto, establecemos {{cssxref ("width")}} en 19.5%, y {{cssxref ("margin-right")}} en 0.625%. Notarás que todo este ancho se suma al 100.625%, lo que haría que el último botón desbordara <ul> y hacerlo caer a la siguiente línea. Sin embargo, lo llevamos de vuelta a 100% utilizando la siguiente regla, que selecciona sólo la última <a> de la lista y elimina el margen. ¡Hecho!
    • Las últimas tres declaraciones son bastante simples y son principalmente solo para fines cosméticos. Centramos el texto dentro de cada enlace, establecemos {{cssxref ("line-height")}} en 3 para dar a los botones algo de altura (que también tiene la ventaja de centrar el texto verticalmente), y establecemos el color del texto a negro.

Nota: Es posible que hayas notado que los elementos de la lista de HTML se colocan en la misma línea entre sí; se hace porque los espacios / saltos de línea entre elementos de bloque en línea crean espacios en la página, al igual que los espacios entre palabras, y dichos espacios romperían nuestro diseño de menú de navegación horizontal. Así que hemos eliminado los espacios. Puede encontrar más información sobre este problema (y sus soluciones) en Luchar contra el espacio entre los elementos de bloque en línea.
