Con los conceptos básicos fuera del camino, vamos a ver más detalladamente los elementos que se utilizan para proporcionar la estructura y el significado de las diferentes partes de un formulario.

Prerrequisitos: Conocimientos básicos de informática y de lenguajes HTML y CSS.
Objetivo: Entender cómo estructurar los formularios HTML y darles semántica para que sean usables y accesibles.

La flexibilidad de los formularios los convierte en una de las estructuras más complejas en HTML; puedes crear cualquier tipo de formulario básico utilizando elementos y atributos destinados a esta función. El uso de la estructura correcta al crear un formulario HTML ayudará a garantizar que el formulario sea utilizable y accesible.

El elemento <form>

El elemento {{HTMLElement ("form")}} define formalmente un formulario y los atributos que determinan el comportamiento del formulario. Cada vez que desees crear un formulario HTML, debes empezar utilizando este elemento y anidando todo el contenido dentro de él. Muchas tecnologías de asistencia y complementos del navegador pueden descubrir elementos {{HTMLElement ("form")}} e implementar ganchos especiales para que sean más fáciles de usar.

Ya lo vimos en el artículo anterior.

Advertencia: Está estrictamente prohibido anidar un formulario dentro de otro formulario. Anidar puede hacer que los formularios se comporten de manera impredecible, por lo que es una mala idea.

Siempre es posible usar un control de formulario fuera de un elemento {{HTMLElement ("form")}}. De forma predeterminada, ese control no tiene nada que ver con ningún formulario a menos que lo asocies con un formulario utilizando el atributo form. Esto se introdujo para permitir vincular explícitamente un control con un formulario incluso si no está anidado dentro de él.

A continuación cubriremos los elementos estructurales que encontrarás anidados en un formulario.

Los elementos <fieldset> y <legend>

El elemento {{HTMLElement ("fieldset")}} es una forma conveniente de crear grupos de widgets que comparten el mismo propósito, con fines de estilo y semánticos. Puedes etiquetar un elemento {{HTMLElement ("fieldset")}} incluyendo un elemento {{HTMLElement ("legend")}} justo debajo de la etiqueta de apertura {{HTMLElement ("fieldset")}}. El contenido de texto de la {{HTMLElement ("leyenda")}} describe formalmente el propósito del {{HTMLElement ("fieldset")}} que está incluido dentro.

Muchas tecnologías de asistencia utilizarán el elemento {{HTMLElement ("leyenda")}} como si fuera parte de la etiqueta de cada control dentro del elemento {{HTMLElement ("fieldset")}} correspondiente. Por ejemplo, algunos lectores de pantalla como Jaws y NVDA leerán el contenido de la leyenda antes de decir la etiqueta de cada control.

Un pequeño ejemplo:

<form>
  <fieldset>
    <legend>Tamaño de zumo de fruta</legend>
    <p>
      <input type="radio" name="size" id="size_1" value="small">
      <label for="size_1">Pequeño</label>
    </p>
    <p>
      <input type="radio" name="size" id="size_2" value="medium">
      <label for="size_2">Mediano</label>
    </p>
    <p>
      <input type="radio" name="size" id="size_3" value="large">
      <label for="size_3">Grande</label>
    </p>
  </fieldset>
</form>

Nota: Puedes encontrar este ejemplo en fieldset-legend.html (verlo en vivo).

Al leer el formulario anterior, un lector de pantalla dirá «Tamaño de zumo de fruta pequeño» para el primer widget, «Tamaño de zumo de fruta mediano» para el segundo y «Tamaño de zumo de fruta grande» para el tercero.

El caso de uso en este ejemplo es uno de los más importantes. Cada vez que tengas un conjunto de botones de opción, debes anidarlos dentro de un elemento {{HTMLElement ("fieldset")}}. Existen otros casos de uso y, en general, el elemento {{HTMLElement ("fieldset")}} también se puede usar para seccionar un formulario. Idealmente, los formularios largos deberían extenderse a través de varias páginas, pero si un formulario se está alargando y debe estar en una sola página, colocar las diferentes secciones relacionadas dentro de diferentes conjuntos de campos mejora la usabilidad.

Debido a su influencia sobre la tecnología de asistencia, el elemento {{HTMLElement ("fieldset")}} es uno de los elementos clave para crear formularios accesibles. Sin embargo, no debes abusar de él. Si es posible, cada vez que crees un formulario, intenta escuchar cómo lo interpreta un lector de pantalla. Si suena extraño, intenta mejorar la estructura del formulario.

El elemento <label>

Como vimos en el artículo anterior, el elemento {{HTMLElement ("label")}} es la forma formal de definir una etiqueta para un widget de formulario HTML. Este es el elemento más importante si deseas crear formularios accesibles: cuando se implementan correctamente, los lectores de pantalla leerán la etiqueta de un elemento de formulario junto con las instrucciones relacionadas, además resulta muy útil para los usuarios videntes. Utiliza este ejemplo que vimos en el artículo anterior:

<label for="name">Nombre:</label> <input type="text" id="name" name="user_name">

Con la etiqueta <label> asociada correctamente con <input> a través de su atributo for (que contiene el atributo id del elemento <input>), un lector de pantalla leerá algo como «Nombre, editar texto».

Hay otra forma de asociar un control de formulario con una etiqueta: anida el control de formulario dentro de <label>, asociándolo implícitamente.

<label for="name">
  Name: <input type="text" id="name" name="user_name">
</label>

Incluso en estos casos, se aconseja establecer el atributo for para garantizar que todas las tecnologías de asistencia comprendan la relación entre la etiqueta y el widget.

Si no hay una etiqueta, o si el control de formulario no está asociado implícita o explícitamente con una etiqueta, un lector de pantalla leerá algo como «Editar texto en blanco», lo cual no es de mucha ayuda.

¡También se puede hacer clic en las etiquetas!

Otra ventaja de configurar correctamente las etiquetas es que puedes hacer clic o activar la etiqueta para activar el widget correspondiente. Esto es útil para controles como entradas de texto, donde puedes hacer clic en la etiqueta y en la entrada para enfocarla, pero es especialmente útil para botones de opción y casillas de verificación: el área de impacto de dicho control puede ser muy pequeña, por lo que es útil para que sea tan fácil de activar como sea posible.

Por ejemplo, al hacer clic en el texto de la etiqueta «Me gustan las cerezas» en el siguiente ejemplo, se alternará el estado seleccionado de la casilla de verificación taste_cherry:

<form>
  <p>
    <input type="checkbox" id="taste_1" name="taste_cherry" value="cherry">
    <label for="taste_1">Me gustan las cerezas</label>
  </p>
  <p>
    <input type="checkbox" id="taste_2" name="taste_banana" value="banana">
    <label for="taste_2">Me gustan los plátanos</label>
  </p>
</form>

Nota: Puedes encontrar este ejemplo en checkbox-label.html (verlo en vivo).

Etiquetas múltiples

Estrictamente hablando, puedes poner varias etiquetas en un solo widget, pero esta no es una buena idea ya que algunas tecnologías de asistencia pueden tener problemas para manejarlas. En caso de poner varias etiquetas, debes anidar un widget y sus etiquetas dentro de un solo elemento {{htmlelement ("label")}}.

Consideremos este ejemplo:

<p> Los campos necesarios se marcan con <abbr title = "required"> * </abbr>. </p>

<!-- Así que esto: -->
<div>
  <label for="username">Nombre:</label>
  <input type="text" name="username">
  <label for="username"><abbr title="required" aria-label="required">*</abbr></label>
</div>

<!-- sería mejor hacerlo así: -->
<div>
  <label for="username">
    <span> Nombre: </span>
    <input id="username" type="text" name="username">
    <abbr title="required" aria-label="required">*</abbr>
  </label>
</div>

<!-- Pero esto es probablemente lo mejor: -->
<div>
  <label for="username">Nombre: <abbr title="required" aria-label="required">*</abbr></label>
  <input id="username" type="text" name="username">
</div>

{{EmbedLiveSample("Multiple_labels", 120, 120)}}

El párrafo de la parte superior establece una regla para los elementos requeridos. La regla debe incluirse antes de usarse para que los usuarios videntes y los usuarios con tecnologías de asistencia, como los lectores de pantalla, sepan lo que significa antes de encontrar un elemento requerido. Si bien esto ayuda a informar a los usuarios lo que significa un asterisco, no se puede confiar en él. Un lector de pantalla pronunciará un asterisco como «estrella» cuando se encuentre uno. Cuando un usuario vidente pasa el ratón por encima, debe aparecer una etiqueta de «requerido», lo cual se logra mediante el uso del atributo title. Los títulos que se leen en voz alta dependen de la configuración del lector de pantalla, por lo que es más confiable incluir también el atributo aria-label, que los lectores de pantalla siempre leen.

Las variantes anteriores aumentan en efectividad a medida que avanzas:

  • En el primer ejemplo, la etiqueta no se lee en absoluto con la entrada; solo se ve «editar texto en blanco», además las etiquetas reales se leen por separado. Los múltiples elementos <label> confunden al lector de pantalla.
  • En el segundo ejemplo, las cosas son un poco más claras: la etiqueta que se lee junto con la entrada es «se requiere el texto de edición del nombre de la estrella», pero las etiquetas aún se leen por separado. Las cosas siguen siendo un poco confusas, pero esta vez funciona un poco mejor porque <input> tiene una etiqueta asociada.
  • El tercer ejemplo es el mejor: la etiqueta se lee en conjunto, y la etiqueta que se lee con la entrada es «nombre requerido editar texto».

Nota: Es posible que obtengas resultados ligeramente diferentes dependiendo de tu lector de pantalla. Esto se probó con VoiceOver (NVDA se comporta de manera similar). Nos encantaría escuchar tus experiencias.

Nota: Puedes encontrar este ejemplo en GitHub como required-labels.html (véalo en vivo). No pruebes el ejemplo con 2 o 3 de las versiones sin comentar: ¡los lectores de pantalla definitivamente se confundirán si tiene múltiples etiquetas y múltiples entradas con el mismo ID!

Estructuras HTML comunes utilizadas en formularios

Más allá de las estructuras específicas de los formularios web, es bueno recordar que el marcado de formularios es solo HTML. Esto significa que puedes usar todo el poder del HTML para estructurar un formulario web.

Como puedes ver en los ejemplos, es una práctica común delimitar una etiqueta y su widget con un elemento {{HTMLElement ("li")}} dentro de una lista {{HTMLElement ("ul")}} o {{HTMLElement ("ol ")}}. Los elementos {{HTMLElement ("p")}} y {{HTMLElement ("div")}} también se usan comúnmente. Se recomiendan las listas para estructurar múltiples casillas de verificación o botones de opción.

Además del elemento {{HTMLElement ("fieldset")}}, también es una práctica común usar títulos HTML (por ejemplo, {{htmlelement ("h1")}}, {{htmlelement ("h2")}}) y seccionar (por ejemplo, {{htmlelement ("section")}}) para estructurar formas complejas.

Sobre todo, depende de ti encontrar un estilo de codificación cómodo que de como resultado formas accesibles y utilizables. Cada sección con funcionalidades diferentes debe estar contenida en un elemento separado {{htmlelement ("section")}}, con elementos {{htmlelement ("fieldset")}} para contener botones de opción.

Aprendizaje activo: construir una estructura de formulario

Pongamos en práctica estas ideas y creemos un formulario un poco más complicado: un formulario de pago. Este formulario contendrá una serie de tipos de control que quizás aún no comprendas. No te preocupes por esto por ahora; en el siguiente artículo descubrirás cómo funcionan (Controles básicos de forma nativa). Por ahora, lee las descripciones cuidadosamente mientras sigues las instrucciones que encontrarás a continuación y empiezas a observar qué elementos de envoltura estamos utilizando para estructurar el formulario y por qué.

  1. Para comenzar, haz una copia local de nuestro archivo de plantilla en blanco y el CSS de nuestro formulario de pago en un nuevo directorio de tu ordenador.
  2. Aplica el CSS al HTML añadiendo la siguiente línea dentro del HTML {{htmlelement ("head")}}: 
    <link href="payment-form.css" rel="stylesheet">
  3. A continuación, crea tu formulario añadiendo el elemento externo {{htmlelement ("form")}}: 
    <form>

</form>
  4. Dentro de las etiquetas <form>>, añade un encabezado y un párrafo para informar a los usuarios cómo se marcan los campos obligatorios: 
    <h1>Forma de pago</h1>
<p> Los campos obligatorios van seguidos de <strong> <abbr title = "required"> * </abbr> </strong>. </p>
  5. A continuación añadimos una sección más grande de código en el formulario, justo debajo de nuestra entrada anterior. Aquí verás que estamos delimitando los campos de información de contacto dentro de un elemento {{htmlelement ("section")}} diferente. Además, tenemos un conjunto de dos botones de opción, cada uno de los cuales colocamos dentro de su propia lista ({{htmlelement ("li")}}) de elementos. También tenemos dos textos estándar {{htmlelement ("input")}} y sus elementos {{htmlelement ("label")}} asociados, cada uno anidado dentro de un elemento {{htmlelement ("p")}} y una casilla para ingresar una contraseña. Añade este código a tu formulario: 
    <section>
    <h2> Información de contacto </h2>
    <fieldset>
      <legend>Título</legend>
      <ul>
          <li>
            <label for="title_1">
              <input type="radio" id="title_1" name="title" value="K" >
              Rei
            </label>
          </li>
          <li>
            <label for="title_2">
              <input type="radio" id="title_2" name="title" value="Q">
              Reina
            </label>
          </li>
          <li> 
            <label for="title_3"> 
              <input type="radio" id="title_3" name="title" value="J">
              Bufón 
            </label> 
          </li>
      </ul>
    </fieldset>
    <p>
      <label for="name">
        <span> Nombre: </span>
        <strong><abbr title="required">*</abbr></strong>
      </label>
      <input type="text" id="name" name="username">
    </p>
    <p>
      <label for="mail">
        <Span> Correo electrónico: </ span>
        <strong><abbr title="required">*</abbr></strong>
      </label>
      <input type="email" id="mail" name="usermail">
    </p>
    <p>
      <label for="pwd">
        <span> Contraseña: </span>
        <strong><abbr title="required">*</abbr></strong>
      </label>
      <input type="password" id="pwd" name="password">
    </p>
</section>
  6. La segunda <sección> de nuestro formulario es la información de pago. Tenemos tres controles diferentes, junto con sus etiquetas, cada uno contenido dentro de un elemento <p>. El primero es un menú desplegable ({{htmlelement ("select")}}) para seleccionar el tipo de tarjeta de crédito. El segundo es un elemento <input> de tipo tel, para ingresar un número de tarjeta de crédito. Si bien podríamos haber usado el tipo de número, no queremos la interfaz de usuario del número. El último es un elemento <input> de tipo fecha, para ingresar la fecha de vencimiento de la tarjeta; este aparecerá con un widget de selector de fecha en navegadores compatibles y volverá a una entrada de texto normal en navegadores no compatibles. Estos tipos de entrada más nuevos se reintroducen en los tipos de entrada HTML5.

    Introduce los siguientes datos debajo de la sección anterior: 
    <section>
    <h2> Información de pago </h2>
    <p>
      <label for="card">
        <span> Tipo de tarjeta: </span>
      </label>
      <select id="card" name="usercard">
        <option value="visa">Visa</option>
        <option value="mc">Mastercard</option>
        <option value="amex">American Express</option>
      </select>
    </p>
    <p>
      <label for="number">
        <span> Número de tarjeta: </span>
        <strong><abbr title="required">*</abbr></strong>
      </label>
      <input type="tel" id="number" name="cardnumber">
    </p>
    <p>
      <label for="date">
        <span> Fecha de caducidad: </span>
        <strong><abbr title="required">*</abbr></strong>
        <em> el formato mm/aa </em>
      </label>
      <input type="date" id="date" name="expiration">
    </p>
</section>
  7. La última sección que añadimos es mucho más simple y contiene solo un {{htmlelement ("botón")}} de tipo submit, para enviar los datos del formulario. Añádelo al final de tu formulario: 
    <p> <button type="submit">Validar el pago </button> </p>
