En el artículo anterior, marcamos un ejemplo de formulario web funcional, presentamos algunos controles de formulario y elementos estructurales comunes, y nos centramos en las mejores prácticas de accesibilidad. A continuación, veremos detalladamente la funcionalidad de los diferentes controles de formulario, o widgets, estudiando todas las diferentes opciones disponibles para recopilar diferentes tipos de datos. En este artículo en particular, veremos el conjunto original de controles de formulario, disponible en todos los navegadores desde los primeros días de la web.

Prerrequisitos: Conocimientos básicos de informática y de lenguajes HTML y CSS.
Objetivo: Comprender en detalle el conjunto original de widgets de formularios nativos disponibles en los navegadores para recopilar datos y cómo implementarlos usando HTML.

Ya conoces algunos elementos de formulario, incluidos {{HTMLelement ('form')}}, {{HTMLelement ('fieldset')}}, {{HTMLelement ('legend')}}, {{HTMLelement ('textarea' )}}, {{HTMLelement ('label')}}, {{HTMLelement ('button')}} y {{HTMLelement ('input')}}. Este artículo cubre:

  • Algunos de los atributos que son comunes en todos los controles de formulario.

Nota: Las características discutidas en este artículo son compatibles con todos los navegadores, lo que no es el caso para todos los controles de formulario. Cubriremos los controles de formularios más nuevos agregados en HTML5 en los próximos dos artículos. Campos de entrada de texto

Los campos de texto {{htmlelement ("input")}} son los widgets de forma más básicos. Son una forma muy conveniente de permitir que el usuario ingrese cualquier tipo de datos, y ya hemos visto algunos ejemplos simples.

Nota: Los campos de texto de formulario HTML son controles simples de entrada de texto sin formato. Esto significa que no puedes usarlos para realizar una edición enriquecida (negrita, cursiva, etc.). Todos los editores de texto enriquecido que encontrarás son widgets personalizados creados con HTML, CSS y JavaScript.

Todos los controles básicos de texto comparten algunos comportamientos comunes:

  • Se pueden marcar como {{htmlattrxref ("readonly", "input")}} (el usuario no puede modificar el valor de entrada pero se envía con el resto de los datos del formulario) o {{htmlattrxref ("disabled", " input ")}} (el valor de entrada no se puede modificar y nunca se envía con el resto de los datos del formulario).
  • Pueden tener un {{htmlattrxref ("placeholder","input")}}; Este es el texto que aparece dentro del cuadro de entrada de texto que debe usarse para describir brevemente el propósito del cuadro.
  • Pueden tener un tamaño limitado (el tamaño físico del cuadro) y longitud máxima (el número máximo de caracteres que se pueden poner en el cuadro).
  • Pueden beneficiarse de corrección ortográfica (utilizando el atributo spellcheck), si el navegador lo admite.

Nota: El elemento {{htmlelement ("input")}} es único entre los elementos HTML porque puede tomar muchas formas diferentes dependiendo de su tipo de valor de atributo Se utiliza para crear la mayoría de los tipos de widgets de formulario, incluidos los campos de texto de una sola línea, controles de fecha y hora, controles sin entrada de texto, como casillas de verificación, botones de opción y selectores de color y botones.

Campos de texto de una sola línea

Se crea un campo de texto de una sola línea utilizando un elemento {{HTMLElement ("input")}} cuyo valor de atributo {{htmlattrxref ("type", "input")}} se establece en text, u omitiendo el atributo {{htmlattrxref ( "type", "input")}} por completo (text es el valor predeterminado). El valor text de este atributo también es el valor de reserva si el navegador no conoce el valor que has especificado para el atributo {{htmlattrxref ("type", "input")}} (por ejemplo, si especificas type="color" y el navegador no admite selectores de color nativos).

Nota: Puedes encontrar ejemplos de todos los tipos de campo de texto de una sola línea en GitHub en single-line-text-fields.html (ver en vivo).

Aquí hay un ejemplo básico de campo de texto de una sola línea:

<input type="text" id="comment" name="comment" value="I'm a text field">

Los campos de texto de una sola línea solo tienen una restricción verdadera: si escribes el texto con saltos de línea, el navegador elimina esos saltos de línea antes de enviar los datos al servidor.

Nota: HTML5 mejoró el campo de texto básico original de una sola línea al agregar valores especiales para el atributo {{htmlattrxref ("type", "input")}} que imponen restricciones de validación específicas y otras características, por ejemplo, específicas para ingresar URL o números. Los cubriremos en el siguiente artículo: Los tipos de entrada HTML5.

Campo de contraseña

Uno de los tipos de entrada originales era el tipo de campo de texto de password:

<input type="password" id="pwd" name="pwd">

El valor de la contraseña no añade restricciones especiales al texto ingresado, pero oscurece el valor ingresado en el campo (por ejemplo, con puntos o asteriscos) para que otros no puedan leerlo fácilmente.

Ten en cuenta que esto es solo una función de interfaz de usuario; a menos que envíes tu formulario de forma segura, se enviará en texto plano, lo que es malo para la seguridad: alguien con malas intenciones podría interceptar tus datos y robar contraseñas, datos de tarjetas de crédito o cualquier otra cosa que hayas enviado. La mejor manera de proteger a los usuarios de esto es alojar cualquier página que involucre formularios a través de una conexión segura (es decir, en una dirección https://), de modo que los datos se cifren antes de enviarse.

Los navegadores reconocen las implicaciones de seguridad de enviar datos de formulario a través de una conexión insegura y tienen advertencias para disuadir a los usuarios de usar formularios inseguros. Para obtener más información sobre lo que implementa Firefox, consulte Contraseñas inseguras.

Contenido oculto

Otro control de texto original es el tipo de entrada hidden. Esto se usa para crear un control de formulario que es invisible para el usuario, pero que aún se envía al servidor junto con el resto de los datos del formulario una vez enviado; por ejemplo, es posible que desees enviar una marca de tiempo al servidor que indique cuándo se realizó un pedido. Debido a que está oculto, el usuario no puede ver ni editar intencionalmente el valor, nunca recibirá el foco y un lector de pantalla tampoco lo notará.

<input type="hidden" id="timestamp" name="timestamp" value="1286705410">

Si creas dicho elemento, es necesario establecer sus atributos de name y value. El valor se puede establecer dinámicamente a través de JavaScript. El tipo de entrada oculta no debe tener una etiqueta asociada.

Otros tipos de entrada de texto, como {{HTMLElement ("input / search", "search")}}, {{HTMLElement ("input / url", "url")}} y {{HTMLElement ("input / tel" , "tel")}}, se añadieron con HTML5. Estos se tratarán en el próximo artículo: tipos de entrada HTML5.

Elementos marcables: casillas de verificación y botones de opción

Los elementos marcables son controles cuyo estado puede cambiar haciendo clic en ellos o en sus etiquetas asociadas. Hay dos tipos de elementos marcables: la casilla de verificación y el botón de opción. Ambos usan el atributo checked para indicar si el widget está marcado de forma predeterminada o no.

Vale la pena señalar que estos widgets no se comportan exactamente como otros widgets de forma. Para la mayoría de los widgets de formulario, una vez que se envía el formulario, se envían todos los widgets que tienen un atributo de name, incluso si no se ha completado ningún valor. En el caso de elementos verificables, sus valores se envían solo si están marcados. Si no están marcados, no se envía nada, ni siquiera su nombre. Si están marcados pero no tienen valor, el nombre se envía con un valor de on.

Nota: Puede encontrar los ejemplos de esta sección en GitHub como checkable-items.html (ver en vivo).

Para una máxima usabilidad/accesibilidad, se recomienda rodear cada lista de elementos relacionados en un {{htmlelement ("fieldset")}}, con un {{htmlelement ("legend")}} que proporcione una descripción general de la lista.  Cada par individual de elementos {{htmlelement ("label")}} / {{htmlelement ("input")}} debe estar contenido en su propio elemento de lista (o similar). El {{htmlelement ('label')}} asociado generalmente se coloca inmediatamente después del botón de opción o casilla de verificación, y sus instrucciones generalmente son el contenido del {{htmlelement ("legend")}} . Mira los ejemplos vinculados anteriormente para ejemplos estructurales.

Casilla de verificación

Se crea una casilla de verificación utilizando el elemento {{HTMLElement ("input")}} con un atributo type establecido en el valor {{HTMLElement ("input / checkbox", "checkbox")}}.

<input type="checkbox" id="carrots" name="carrots" value="carrots" checked>

Al incluir el atributo checked, la casilla de verificación se marca automáticamente cuando se carga la página. Al hacer clic en la casilla de verificación o en su etiqueta asociada, se activa y desactiva la casilla de verificación.

Debido a la naturaleza de encendido y apagado de las casillas de verificación, se considera un botón de alternancia, con muchos desarrolladores y diseñadores ampliando el estilo de casilla de verificación predeterminado para crear botones que parecen interruptores de alternancia. Boton de opción

Se crea un botón de opción usando el elemento {{HTMLElement ("input")}} con su atributo {{htmlattrxref ("type", "input")}} establecido en el valor radio:

<input type="radio" id="soup" name="meal" checked>

Se pueden unir varios botones de opción. Si comparten el mismo valor para su atributo {{htmlattrxref ("name", "input")}}, se considerará que están en el mismo grupo de botones. Solo se puede verificar un botón en un grupo dado a la vez. Esto significa que cuando uno de ellos se marca, todos los demás se desmarcan automáticamente. Cuando se envía el formulario, solo se envía el valor del botón de opción marcado. Si ninguno de ellos está marcado, se considera que el conjunto completo de botones de opción está en un estado desconocido y no se envía ningún valor con el formulario. Una vez que se marca uno de los botones de opción en un grupo de botones con el mismo nombre, no es posible desmarcar todos los botones sin restablecer el formulario.

<fieldset>
  <legend> ¿Cuál es su comida favorita? </legend>
  <ul>
    <li>
      <label for="soup">Sopa</label>
      <input type="radio" id="soup" name="meal" value="soup" checked>
    </li>
    <li>
      <label for="curry">Curry</label>
      <input type="radio" id="curry" name="meal" value="curry">
    </li>
    <li>
      <label for="pizza">Pizza</label>
      <input type="radio" id="pizza" name="meal" value="pizza">
    </li>
  </ul>
</fieldset>

Botones reales

El botón de opción no es realmente un botón, a pesar de su nombre; ¡sigamos y miremos los botones reales! Hay tres tipos de entrada dependiendo de los botones que se utilicen:

enviar
Envía los datos del formulario al servidor. Para los elementos {{HTMLElement ("button")}}, omitir el atributo type (o un valor de tipo no válido) da como resultado un botón de envío.
reiniciar
Restablece todos los controles de formulario a sus valores por defecto.
botón
Botones que no tienen efecto automático pero que se pueden personalizar con código JavaScript.

Además, el elemento {{htmlelement ("button")}} puede tomar un atributo type para imitar estos tres tipos de entrada La principal diferencia entre los dos es que los elementos <button> reales son mucho más estilizables.

Nota: El tipo de entrada de imagen también se representa como un botón. Cubriremos este tema más adelante.

Nota: Puedes encontrar los ejemplos de esta sección en GitHub como button-examples.html (ver en vivo).

A continuación puedes encontrar ejemplos de cada tipo de botón <input>, junto con el tipo de botón equivalente.

enviar

<button type="submit">
    Este es un botón <strong>Enviar</strong>
</button>

<input type="submit" value="Este es un botón de enviar">

reiniciar

<button type="reset">
    Se trata de un botón <strong>Reiniciar</strong>
</button>

<input type="reset" value="Este es un botón de reinicio">

anónimo

<button type="button">
    Se trata de un botón <strong>anónimo</strong>
</button>

<input type="button" value="Este es un botón anónimo">

Los botones siempre se comportan igual, independientemente de su usas un elemento {{HTMLElement ("button")}} o un elemento {{HTMLElement ("input")}}. Sin embargo, como puedes ver en los ejemplos, los elementos {{HTMLElement ("button")}} te permiten usar HTML en tu contenido, que se inserta entre las etiquetas <button> de apertura y cierre. Los elementos {{HTMLElement ("input")}} por otro lado son elementos vacíos; el contenido que se muestra se inserta dentro del atributo value y, por lo tanto, solo acepta texto sin formato.

Los siguientes ejemplos muestran los tipos de entrada de botones predeterminados, enfocados y deshabilitados: en Firefox 71 y Safari 13 en macOS, y Chrome 79 y Edge 18 en Windows 10.

Botón de imagen

El control del botón de imagen se representa exactamente como un elemento {{HTMLElement ("img")}}, excepto que cuando el usuario hace clic en él, se comporta como un botón de envío.

Se crea un botón de imagen usando un elemento {{HTMLElement ("input")}} con su atributo {{htmlattrxref ("type", "input")}} establecido en el valor image. Este elemento admite exactamente el mismo conjunto de atributos que el elemento {{HTMLElement ("img")}}, además de todos los atributos admitidos por otros botones de formulario.

<input type="image" alt="Click me!" src="my-img.png" width="80" height="30">

Si el botón de imagen se usa para enviar el formulario, este control no envía su valor; en cambio, se envían las coordenadas X e Y del clic en la imagen (las coordenadas son relativas a la imagen, lo que significa que la esquina superior izquierda la esquina de la imagen representa la coordenada (0, 0)). Las coordenadas se envían como dos pares clave/valor:

  • La clave del valor X es el valor del atributo {{htmlattrxref ("name", "input")}} seguido de la cadena «.x».
  • La clave del valor Y es el valor del atributo {{htmlattrxref ("name", "input")}} seguido de la cadena «.y».

Por ejemplo, cuando haces clic en la imagen en la coordenada (123, 456) y se envía a través del método get , verás los valores añadidos a la URL de la siguiente manera:

http://foo.com?pos.x=123&pos.y=456

Esta es una forma muy conveniente de construir un «mapa dinámico». La forma en que se envían y recuperan estos valores se detalla en el artículo Enviar datos del formulario.

Selector de archivos

Hay un último tipo de <input> que nos llegó a principios del HTML: la entrada de archivo. Los formularios pueden enviar archivos a un servidor (esta acción específica también se detalla en el artículo Enviar de datos del formulario). El widget selector de archivos se puede usar para elegir uno o más archivos para enviar.

Para crear un widget selector de archivos, utiliza el elemento {{HTMLElement ("input")}} con su atributo {{htmlattrxref ("type", "input")}} establecido en file. Los tipos de archivos que se aceptan pueden restringirse utilizando el atributo {{htmlattrxref ("accept", "input")}}. Además, si quieres permitir que el usuario elija más de un archivo, puedes hacerlo agregando el atributo {{htmlattrxref ("multiple", "input")}}.

Ejemplo

En este ejemplo, se crea un selector de archivos que solicita archivos de imágenes gráficas. El usuario puede seleccionar múltiples archivos en este caso.

<input type="file" name="file" id="file" accept="image/*" multiple>

En algunos dispositivos móviles, el selector de archivos puede acceder a fotos, videos y audio capturados directamente por la cámara y el micrófono del dispositivo agregando información de captura al atributo accept de la siguiente manera:

<input type="file" accept="image/*;capture=camera">
<input type="file" accept="video/*;capture=camcorder">
<input type="file" accept="audio/*;capture=microphone">

Atributos comunes

Muchos de los elementos utilizados para definir controles de formulario tienen sus propios atributos específicos. Sin embargo, hay un conjunto de atributos comunes para todos los elementos de formulario. Ya has conocido algunos, pero a continuación encontrarás una lista de esos atributos comunes para futuras referencias::

Nombre del atributo Valor por defecto Descripción
autofocus falso Este atributo booleano te permite especificar que el elemento debe tener automáticamente el foco de entrada cuando se carga la página. Solo un elemento asociado a un formulario en un documento puede tener este atributo especificado.
disabled falso Este atributo booleano indica que el usuario no puede interactuar con el elemento. Si no se especifica este atributo, el elemento hereda su configuración del elemento contenedor, por ejemplo, {{HTMLElement ("fieldset")}}. Si no hay ningún elemento contenedor con el conjunto de atributos disabled, el elemento está habilitado.
form El elemento <form> con el que está asociado el widget, usado si no está anidado dentro de ese formulario. El valor del atributo debe ser el atributo id de un elemento {{HTMLElement ("form")}} en el mismo documento. Esto te permite asociar un control de formulario con un formulario que esté fuera, incluso si está dentro de un elemento de formulario diferente.
name El nombre del elemento se envía con los datos del formulario.
value El valor inicial del elemento.
