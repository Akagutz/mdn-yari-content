En el segundo artículo de este módulo, analizamos algunas características más avanzadas de las tablas HTML, como subtítulos/resúmenes y agrupamos sus filas en secciones de encabezado, cuerpo y pie de página, así como también analizamos la accesibilidad de las tablas para usuarios con discapacidad visual.

Prerrequisitos: Conceptos básicos de HTML (ver Introducción al HTML).
Objetivo: Obtener información sobre las características más avanzadas de tablas HTML y la accesibilidad de las tablas.

Añadir un subtítulo a tu tabla con <caption>

Puedes darle un título a tu tabla colocándolo dentro de un elemento {{htmlelement ("caption")}} y anidándolo dentro del elemento {{htmlelement ("table")}}. Debe ponerlo justo debajo de la etiqueta de apertura <table>.

<table>
  <caption> Dinosaurios en el período Jurásico </caption>

  ...
</table>

Como puedes deducir del breve ejemplo anterior, el título debe contener una descripción del contenido de la tabla. Esto es útil para todos los lectores que deseen descubrir rápidamente si la tabla les resulta útil mientras escanean la página, pero básicamente es para usuarios con visibilidad reducida. En lugar de que un lector de pantalla lea el contenido de muchas celdas solo para averiguar de qué trata la tabla, puede confiar en un título y luego decidir si leer la tabla con mayor detalle.

Se coloca un subtítulo directamente debajo de la etiqueta <table>.

Nota: El atributo {{htmlattrxref("resumen","tabla")}} también se puede usar en el elemento table para proporcionar una descripción; los lectores de pantalla también lo leen. Sin embargo, recomendamos usar el elemento caption, ya que el resumen está {{glossary("obsoleto")}} según la especificación HTML5, y no puede ser leído por los usuarios (no aparece en la página).

  1. Abre el horario escolar del profesor de idiomas desde el final de HTML Table Basics, o haz una copia local de nuestro archivo timetable-fixed.html.
  2. Añadir un título adecuado para la mesa.
  3. Guarda tu código y ábrelo en un navegador para ver cómo se ve.

Añadir estructura con <thead>, <tfoot> y <tbody>

A medida que la estructura de las tablas se vuelven un poco más compleja, es útil darles una estructura más definida. Una forma clara de hacerlo es mediante {{htmlelement ("thead")}}, {{htmlelement ("tfoot")}} y {{htmlelement ("tbody")}}, que te permiten marcar un encabezado, pie de página y una sección del cuerpo de la tabla.

Estos elementos no hacen que la tabla sea más accesible para los usuarios de lectores de pantalla y no resultan en ninguna mejora visual por sí mismos. Sin embargo, son muy útiles para el estilo y el diseño, ya que actúan como ganchos útiles para agregar CSS a su tabla. Para darte algunos ejemplos interesantes, en el caso de una tabla larga, puedes hacer que el encabezado y el pie de página se repitan en cada página impresa, así como también que el cuerpo de la tabla se muestre en una sola página y tenga los contenidos disponibles desplazándose hacia arriba y hacia abajo.

Para utilizarlos:

  • El elemento <thead> debe delimitar el encabezado de la tabla; esta suele ser la primera fila que contiene los encabezados de las columnas, pero no siempre es así. Si estás utilizando los elementos {{htmlelement ("col")}} / {{htmlelement ("colgroup")}}, el encabezado de la tabla debe estar justo debajo.
  • El elemento <tfoot> delimita la parte de la tabla correspondiente al pie de página; esta podría ser una fila final con elementos en las filas anteriores. Puedes incluir el pie de página de la tabla justo en la parte inferior o justo debajo del encabezado (el navegador aún lo mostrará en la parte inferior de la tabla).
  • El elemento <tbody> delimita las otras partes del contenido de la tabla que no están en el encabezado o pie de página de la tabla. Aparecerá debajo del encabezado de la tabla o, a veces, en el pie de página, según cómo hayas decidido estructurarlo.

Nota: <tbody> se incluye siempre en todas las tablas de forma implñicita si no lo especificas en tu código. Para comprobarlo, abre uno de tus ejemplos anteriores que no incluya <tbody> y mira el código HTML en las herramientas de desarrollo de tu navegador; verás que el navegador ha agregado esta etiqueta. Quizás te preguntes por qué debería molestarte en incluirlo. Debes hacerlo para tener más control sobre la estructura y el estilo de la tabla.

Anidar tablas

Es posible anidar una tabla dentro de otra, siempre que incluyas la estructura completa, incluido el elemento <table>. Por lo general, esto no se recomienda, ya que hace que el marcado sea más confuso y menos accesible para los usuarios de lectores de pantalla, y en muchos casos también podría insertar celdas/filas/columnas adicionales en la tabla existente. Sin embargo, a veces es necesario, por ejemplo, si deseas importar contenido de forma sencilla desde otras fuentes.

El siguiente marcado muestra una tabla anidada simple:

<table id="tabla1">
  <tr>
    <th>título1</th>
    <th>título2</th>
    <th>título3</th>
  </tr>
  <tr>
    <td id="nested">
      <table id="tabla2">
        <tr>
          <td>celda1</td>
          <td>celda2</td>
          <td>celda3</td>
        </tr>
      </table>
    </td>
    <td>celda2</td>
    <td>celda3</td>
  </tr>
  <tr>
    <td>celda4</td>
    <td>celda5</td>
    <td>celda6</td>
  </tr>
</table>

La salida se verá así:

título1 título2 título3
celda1 celda2 celda3
celda2 celda3
celda4 celda5 celda6

Tablas para usuarios con discapacidad visual

Repasemos brevemente cómo usamos las tablas de datos. Una tabla puede ser una herramienta útil para darnos un acceso rápido a los datos y permitirnos buscar diferentes valores. Por ejemplo, echa un vistazo a la tabla que encontrarás a continuación para saber cuántos anillos se vendieron en Gante el agosto pasado. Para comprender su información, hacemos asociaciones visuales entre los datos de esta tabla y sus encabezados de columna y/o fila.

Pero, ¿y si no puedes hacer esas asociaciones visuales? ¿Cómo se podría leer una tabla como la anterior? Las personas con discapacidad visual a menudo usan un lector de pantalla que les lee la información de las páginas web. Esto no resulta un problema cuando estás leyendo un texto sin formato, pero interpretar una tabla puede ser un gran desafío para una persona ciega. Sin embargo, con el marcado adecuado podemos reemplazar las asociaciones visuales por otras programáticas.

Nota: Había alrededor de 253 millones de personas con discapacidad visual según los en 2017.

Esta sección del artículo proporciona técnicas adicionales para hacer que las tablas sean lo más accesibles posible.

Usar encabezados de columna y fila

Los lectores de pantalla identificarán todos los encabezados y los usarán para hacer asociaciones programáticas entre esos encabezados y las celdas con las que se relacionan. La combinación de encabezados de columna y fila identificará e interpretará los datos en cada celda para que los usuarios de lectores de pantalla puedan interpretar la tabla de manera similar a como lo hace un usuario vidente.

Ya cubrimos los encabezados en nuestro artículo anterior; consulte Añadir encabezados con elementos <th>.

El atributo scope

Un nuevo tema para este artículo es el atributo {{htmlattrxref ("scope", "th")}}, que se puede agregar al elemento <th> para indicar a los lectores de pantalla exactamente para qué celdas es el encabezado. Mirando hacia atrás a nuestro ejemplo de registro de gastos de antes, podrías definir inequívocamente los encabezados de columna como encabezados de columna de esta forma:

<thead>
  <tr>
    <th scope="col">Compra</th>
    <th scope="col">Ubicación</th>
    <th scope="col">Fecha</th>
    <th scope="col">Evaluación</th>
    <th scope="col">Coste (€)</th>
  </tr>
</thead>

Y cada fila podría tener un encabezado definido de esta manera (si añadimos encabezados de fila y encabezados de columna):

<tr>
  <th scope="row"> corte de pelo</th>
  
<td> Peluquería </td>
  <td>12/09</td>
  
<td> Gran idea </td>
  <td>30</td>
</tr>

Los lectores de pantalla reconocerán el marcado estructurado de esta manera y permitirán a sus usuarios leer toda la columna o fila a la vez, por ejemplo:

scope tiene dos valores posibles más: colgroup y rowgroup. Estos se utilizan para encabezados que se encuentran sobre la parte superior de varias columnas o filas. Si vuelves a echar un vistazo a la tabla «Artículos vendidos en agosto de 2016» al comienzo de esta sección, verás que la celda «Ropa» se encuentra encima de las celdas «Pantalones», «Faldas» y «Vestidos» Todas estas celdas deben estar marcadas como encabezados (<th>), pero «Ropa» es un encabezado que se encuentra en la parte superior y define los otros tres subtítulos. Por lo tanto, «Ropa» debería obtener un atributo de scope="colgroup", mientras que los demás obtendrían un atributo de scope="col".

Los atributos de id y encabezados

Una alternativa al uso del atributo scope es usar los atributos {{htmlattrxref ("id")}} y {{htmlattrxref ("headers", "td")}} para crear asociaciones entre encabezados y celdas. La forma en que se usan es la siguiente:

  1. Añades un id único a cada elemento <th>>.
  2. Añades un atributo headers a cada elemento <td>. Cada atributo headers debe contener una lista de losid de todos los elementos <th> que actúan como encabezado de esa celda, separados por espacios.

Esto le da a tu tabla HTML una definición explícita de la posición de cada celda en la tabla, definida por los encabezados de cada columna y fila de la que forma parte, como una hoja de cálculo. Para que funcione bien, la tabla realmente necesita encabezados de columna y fila.

Volviendo a nuestro ejemplo de gastos, los dos fragmentos anteriores podrían reescribirse así:

<thead>
  <tr>
    <th id="purchase">Compra </th>
    <th id="location">Ubicación </th>
    <th id="date">Fecha</th>
    <th id="evaluation">Evaluación </th>
    <th id="cost">Coste (€) </th>
  </tr>
</thead>
<tbody>
<tr>
  <th id="haircut">Corte de pelo </th>
  <td headers="location haircut">Peluquería </td>
  <td headers="date haircut">12/09</td>
  <td headers="evaluation haircut">Gran idea </td>
  <td headers="cost haircut">30</td>
</tr>

  ...

</tbody>

Nota: Este método crea asociaciones muy precisas entre los encabezados y las celdas de datos, pero utiliza mucho más marcado y no deja espacio para errores.  El enfoque scope suele ser suficiente para la mayoría de las tablas.

  3. Por último, intenta hacer otra copia de los archivos de inicio, y esta vez haga que la tabla sea más accesible utilizando los atributos id y headers.
