The HTML <footer> element represents a footer for its nearest sectioning content or sectioning root element. A footer typically contains information about the author of the section, copyright data or links to related documents.

Sisältökategoriat Virtaussisältö, palpable content.
Sallittu sisältö Virtaussisältö, but with no <footer> or {{HTMLElement("header")}} descendants.
Tagin pois jättäminen {{no_tag_omission}}
Saliitut vanhemmat Any element that accepts virtaussisältö. Note that a <footer> element must not be a descendant of an {{HTMLElement("address")}}, {{HTMLElement("header")}} or another <footer> element.
Sallitut ARIA roolit {{ARIARole("group")}}, {{ARIARole("presentation")}}
DOM rajapinta {{domxref("HTMLElement")}}

Ominaisuudet

Tämä elementti sisältää vain globaalit ominaisuudet.

Käyttöohjeet

  • Enclose information about the author in an {{HTMLElement("address")}} element that can be included into the <footer> element.
  • The <footer> element is not sectioning content and therefore doesn't introduce a new section in the outline.

Esimerkkejä

<footer>
  Some copyright info or perhaps some author
  info for an &lt;article&gt;?
</footer>

Koskien esteettömyyttä

The VoiceOver screen reader has an issue where the footer landmark role is not announced in the landmark rotor. To address this, add role="contentinfo" to the footer element.

  • WebKit Bugzilla: 146930 – AX: HTML native elements (header, footer, main, aside, nav) should work the same as ARIA landmarks, sometimes they don't
