Dans notre module de programmation côté serveur, nous répondrons à quelques questions fondamentales sur la programmation côté serveur. "Qu'est-ce que c'est ?", "En quoi diffère t-elle de la programmation côté client ?" et "Pourquoi est-ce si utile ?". Nous fournirons ensuite un aperçu de certaines infrastructures d'applications (aussi appelé frameworks) web côté serveurs les plus populaires, ainsi que des conseils sur la façon de selectionner celui qui sera le plus approprié pour créer votre premier site. Enfin nous vous fournirons un article d'introduction concernant la sécurité d'un serveur web.   

 

Prérequis

Avant de commencer ce module, vous n'avez pas besoin d'avoir des connaissances en programmation web côté serveur ou tout autre type de programmation.

Vous aurez besoin de comprendre "comment fonctionne le web". Nous vous recommandons de lire d'abord les sujets suivants:

  • Qu'est-ce qu'un serveur web
  • De quel logiciel ai-je besoin pour créer un site Web ?
  • Comment transférer des fichiers sur un serveur Web ?

Avec cette compréhension de base, vous serez prêts à parcourir les modules de cette section.

Guides

Introduction au côté serveur(anglais)
Bienvenue au cours de programmation débutant de MDN! Dans ce premier article, nous examinerons la programmation côté serveur , répondant à des questions comme «En quoi consiste t-elle?», «En quoi diffère t-elle de la programmation côté client?» et «Pourquoi est-ce si utile? Après avoir lu cet article, vous comprendrez la puissance supplémentaire disponible pour les sites Web via la programmation côté serveur.
Présentation client-serveur(anglais)
Maintenant que vous connaissez le but et les avantages potentiels de la programmation côté serveur, nous allons examiner en détail ce qui se passe lorsqu'un serveur reçoit une "requête dynamique" d'un navigateur. Comme la plupart des codes côté serveur traitent les demandes et les réponses de la même manière, cela vous aidera à comprendre ce que vous devez faire lorsque vous écrivez votre propre code.
Frameworks web côté serveur(anglais)
Le dernier article vous a montré ce qu'une application Web côté serveur doit faire pour répondre aux demandes d'un navigateur Web. Nous allons maintenant vous montrer comment les frameworks web peuvent simplifier ces tâches et vous aider à choisir le bon framework pour coder votre première application web côté serveur.
Sécurité de votre site web

La sécurité de votre site Web requiert une vigilance dans tous les aspects de sa conception et de son utilisation. Cet article d'introduction ne fera pas de vous un gourou de la sécurité des sites Web, mais il vous aidera à comprendre les premières mesures importantes que vous pouvez prendre pour renforcer votre application Web contre les menaces les plus courantes.

Évaluations

 

Ce module "aperçu" n'a aucune évaluation car nous ne vous avons pas encore montré de code. Nous espérons à ce stade que vous avez une bonne compréhension des types de fonctionnalités que vous pouvez fournir en utilisant la programmation côté serveur et que vous avez pris une décision quant à l'infrastructure Web côté serveur que vous utiliserez pour créer votre premier site Web.
