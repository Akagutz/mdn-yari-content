Les articles ci-dessous vous aideront à obtenir le code source de Mozilla, à naviguer dans ce code et à savoir comment vérifier dans l'arborescence les changements que vous proposez.

Thèmes de la Documentation
Obtenir le code depuis le dépôt Mercurial
Si vous prévoyez de contribuer au projet Mozilla, la meilleure façon d'obtenir le code est de le vérifier à partir du référentiel de contrôle de version. Apprenez à faire cela ici.
Obtenir une machine virtuelle pré-configurée de Mozilla build system
Ceci est le moyen le plus facile pour commencer : utiliser une machine virtuelle VirtualBox qui est déjà configurée avec un environnement complet pour vous. Il suffit de démarrer la machine virtuelle et de commencer votre devéloppement !
Téléchargement du code en utilisant HTTP ou FTP
Si vous voulez récupérer le code pour une version spécifique d'un produit particulier de Mozilla, vous préférerez peut-être télécharger une archive du code source.
Visualisation et la recherche du code source de Mozilla en ligne
Apprenez à utiliser MXR, outil de recherche et de navigation en ligne de Mozilla pour l'accès au code source. Ce ne sont pas une bonne façon de télécharger le code, mais il est un excellent moyen de le fouiller.
Navigation dans le code source de Mozilla (page en Anglais)
Renseignez-vous sur les différents dossiers dans l'arborescence source de Mozilla, et comment trouver ce que vous cherchez.
Bugs pour les nouveaux arrivants
Si vous êtes nouveau dans le projet et que vous voulez quelque chose à travailler, regardez ici.
Mozilla Coding Style Guide (page en Anglais)
Le guide de style de code fournit des informations sur la façon dont vous devez formater votre code source pour vous assurer que les personnes en charge de vérifier votre code ne se moqueront pas de vous.
Guide de développement d'interface (page en Anglais)
Lignes directrices et de la documentation sur la façon de créer et de mettre à jour des interfaces XPCOM.
Try Servers (Wiki Mozilla - page en Anglais)
Produits Mozilla developpés sur au moins trois plateformes. Si vous ne disposez pas d'accès à tous, vous pouvez utiliser Try Servers (Serveur d'essai) afin de tester vos patchs et assurez-vous les tests fonctionnent.
Création d'un patch (page en Anglais)
Une fois que vous avez fait une modification du code de Mozilla, la prochaine étape (après s'être assuré que cela fonctionne) est de créer un patch et de le soumettre pour examen. Cet article doit être entièrement mis à jour pour Mercurial.
Obtenir l'access Commit au code source (page en Anglais)
Sentez-vous prêt à rejoindre quelques-uns, les orgueilleux, les committers ? Découvrez comment obtenir l'accès au code Mozilla.
Obtenir l'ancien code de Mozilla CVS
Les anciennes versions du code source de Mozilla, ainsi que les versions actuelles de NSS et NSPR, sont conservés dans un dépôt CVS. Renseignez-vous sur comment les obtenir dans cet article.
