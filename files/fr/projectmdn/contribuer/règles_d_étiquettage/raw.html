Une des fonctionnalités importantes de MDN qui aide les utilisateurs à trouver du contenu sont les étiquettes. Chaque page peut ne pas être étiquetée ou avoir plusieurs étiquettes (au moins une serait un minimum) afin d'aider à catégoriser le contenu. Il existe de nombreuses manières d'aider à organiser l'information sur MDN. Cette page va vous aider afin de savoir étiqueter efficacement les pages afin de conserver l'organisation du contenu.

Pour un guide concernant l'interface utilisateur pour modifier les étiquettes sur les pages, référez-vous à la section étiquetage dans notre guide du contributeur.

Usage des étiquettes sur MDN

Les étiquettes sont utilisées pour différentes raisons sur MDN :

Catégorisation
Quel type de document est-ce ? Est-ce une référence ? Un tutoriel ? Une page d'accueil ?
Identification du sujet
De quel sujet cet article parle-t-il ? Est-ce à propos d'une API ? Du DOM ?
État d'avancement d'une technologie
Quel est l'avancement de la technologie ? Est-elle non standardisée ? Obsolète ou dépréciée ? Expérimentale ?
Niveau de difficulté
Pour les tutoriels et les guides, quel est le niveau de difficulté du sujet abordé par l'article ?

Guide des types d'étiquettes

Voici un guide rapide des différents types d'étiquettes existants et de valeurs possibles pour ceux-ci.

Catégorie

L'étiquetage d'un article avec une ou plusieurs des catégories suivantes facilitera la création de page d'accueil, de tables des matières. Ces termes seront aussi utilisés par notre nouveau système de recherche afin de permettre à l'utilisateur de trouver la référence ou le guide de leur choix.

Les noms de catégories suivants sont des termes standards sur MDN.

Référence
L'article est un contenu de référence à propos d'un API, d'un élément, d'un attribut, d'une propriété ou autre sujet semblable.
Accueil
Cette page est une page d'accueil.
Guide
Cet article est un guide.

Sujet

En identifiant le sujet abordé par l'article, vous aidez également à l'amélioration des résultats de recherches ainsi que la qualité des pages d'accueil et d'autres aides à la navigation.

Il existe une certaine flexibilité dans cette liste, puisque de nouveaux sujets peuvent être identifiés. Toutefois, nous tenons à ne compter dans cette liste que des noms d'APIs ou de technologies. Voici quelques exemples :

  • HTML
  • CSS
  • JavaScript (remarquez le S majuscule !)
  • Document
  • DOM
  • API (pour chaque interface, méthode, ou propriété)
  • Graphismes
  • Firefox OS
  • Gecko
  • XUL
  • XPCOM
  • SVG
  • WebGL
  • Élément
  • Node

En général, le nom d'une interface qui possède un certain nombre de pages liées, tel que Node (qui possède de nombreuses pages pour ses propriétés et méthodes) est un bon exemple d'étiquette utile, tout comme l'est le nom d'une technologie. Une page à propos de WebGL peut être étiquetée Graphismes et WebGL par exemple, alors qu'une page à propos de {{HTMLElement("canvas")}} peut être étiquetée avec HTML, Élément, Canvas et Graphismes.

État d'avancement d'une technologie

Afin d'aider le lecteur à comprendre le niveau de maturité d'une technologie, nous utilisons les étiquettes pour mentionner le niveau d'avancement de la spécification de ladite technologie. Nous n'allons pas jusque dans le détail de la spécification et où elle en est dans le processus (c'est le rôle du tableau des spécifications), mais cela aide le lecteur à juger en un clin d'œil s'il est raisonnable d'utiliser la technologie ou non.

Voici les valeur possibles pour ces étiquettes :

Non-standard
Indique que la technologie ou l'API décrite sur la page ne fait pas partie d'un standard, mais est considérée stable dans le ou les navigateurs qui la prennent en charge. Si cette étiquette n'est pas utilisée, il est supposé que l'article couvre un sujet standardisé. Le tableau de compatibilité sur la page doit confirmer quels navigateurs prennent en charge l'API ou la technologie.
Déprécié
La technologie ou l'API abordée sur cette page a été marquée comme déprécie dans la spécification, et il est attendu qu'elle soit un jour retirée, mais est généralement accessible pour les navigateurs actuels.
Obsolète
La technologie ou l'API a été jugée obsolète et a été retirée (ou est en cours de retrait) de tous ou de la plupart des navigateurs actuels.
Expérimental
cette technologie n'est pas standardisée, et expérimentale. Elle peut devenir ou non partie d'un standard dans la futur.
Privilèges nécessaires
Cette API requiert un accès privilégié à l'appareil sur lequel le code est exécuté.
Certifié seulement
Cette API ne fonctionne qu'avec du code certifié.

Quelle que soit l'utilisation de ces étiquettes, assurez-vous d'inclure un tableau de compatibilité dans votre article !

Niveau de difficulté

L'étiquette de niveau de difficulté est seulement utilisée pour les guides et tutoriels (c'est-à-dire les pages étiquettées par Guide). Ces étiquettes permettent aux lecteurs de lire des tutoriels correspondant à leur niveau de confort vis-à-vis d'une technologie. Trois valeurs existent pour ceci :

Débutant
Les articles conçus comme des introductions à la technologie pour des lecteurs n'ayant jamais utilisé ou ayant une connaissance surperficielle du sujet abordé.
Intermédiaire
Les articles conçus pour des utilisateurs ayant débuté leur utilisation de la technologie mais qui ne sont pas des experts en la matière.
Avancé
Les articles visant à approfondir les connaissances du lecteur sur un sujet donné.

En conclusion

Avec ces différentes sortes d'étiquettes, vous pouvez créer un ensemble permettant de décrire efficacement le contenu d'une page. Voici quelques exemples :

Un tutoriel à propos de WebGL pour les débutants
Les étiquettes appropriées seraient : WebGL, Graphismes, Guide, Débutant
Une page de référence pour l'élément {{HTMLElement("canvas")}}
Cette page devrait avoir les étiquettes suivantes : Canvas, HTML, Élément, Graphismes, Référence
Une page d'accueil pour les outils de développement de Firefox OS
Cette page devrait être étiquetée avec Outils, Firefox OS, Accueil
