Le header (en-tête) Connection indique si la connection réseau doit rester ouverte après la fin de la transaction courante. Si la valeur est   keep-alive (reste en vie),  la connection sera persistante et ne sera pas clôturée, ainsi les requêtes suivantes utiliseront cette même connection.

Les headers relatifs aux connections comme Connection ne doivent pas être utilisé avec HTTP/2.

Excepté pour les headers hop-by-hop standards  ({{HTTPHeader("Keep-Alive")}}, {{HTTPHeader("Transfer-Encoding")}}, {{HTTPHeader("TE")}}, {{HTTPHeader("Connection")}}, {{HTTPHeader("Trailer")}}, {{HTTPHeader("Upgrade")}}, {{HTTPHeader("Proxy-Authorization")}} and {{HTTPHeader("Proxy-Authenticate")}}), tous les headers hop-by-hop utilisés par le message doivent être listés dans le header Connection, ainsi le premier proxy sait qu'il doit les consommer et qu'il ne doit pas le transmettre au noeud suivant. Les headers hop-by-hop stantards peuvent aussi être listés (c'est souvent le cas de {{HTTPHeader("Keep-Alive")}}, mais cela n'est pas obligatoire).

Header type {{Glossary("General header")}}
{{Glossary("Forbidden header name")}} oui

Syntaxe

Connection: keep-alive
Connection: close

Directives

close
Inique que soit le client ou le serveur souhaite fermer la connection. C'est la valeur par défaut pour les requêtes HTTP/1.0.
tout header HTTP, séparé par une virgule [en général, uniquement keep-alive]
Indique que le client souhaite garder la connection ouverte. C'est la valeur par défaut pour les requêtes HTTP/1.1. La liste des headers contient les noms des headers qui doivent être supprimés par le premier proxy non-transparent: ces headers définissent la connection entre l'émetteur et le premier noeud, pas le noeud final.
