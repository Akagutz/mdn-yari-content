概要

MDN Web Docs は Web 開発者向けの最高のテクニカルリファレンスで、毎月700万人を超える世界中のユーザーと1,000人のボランティアのコンテンツ寄稿者が、より良い Web を作るために取り組んでいます。Web 開発者は MDN が大好きで、私たちは開発者のために可能な限り最高のコンテンツを提供し続けることができる、Web 開発者のバックグラウンドを持つ経験豊富なテクニカルライター (またはテクニカルライティングのバックグラウンドを持つ We b開発者) を探しています。

テクニカルライターは複雑な情報を受け取り、それを明確かつ正確に伝達し、開発者の考えや必要な情報を理解し、エンジニアとうまく連携し、ソフトウェア開発ツールとプラクティスを深く理解しています。この役割では、テクニカルライターは、Web 開発およびオープン Web に関連する技術情報を文書化し、コードサンプルとチュートリアルを作成し、Web 標準および開発中のテクノロジの状態について最新情報を提供します。

MDN テクニカルライターは、製品およびプラットフォームの開発に関係するステークホルダーとしても参加しており、Web に情熱を傾けている世界規模の貢献者コミュニティと協力しています。MDN テクニカルライターは開発者としての経験 (Webが望ましい) があり、コード (主に JavaScript、HTML、および CSS) を読み書きできます。

責任

  • 参考資料、ガイド、チュートリアルなど、開発者向けの技術文書とリソースを作成します
  • サンプルコードとデモを作成します
  • 文書化された Web 標準、内部プロジェクトのドキュメントとバグ、そしてプロジェクトエンジニアとチームメンバーへのインタビューからソース情報を入手する。
  • 技術的および編集上のレビューを求め、フィードバックを組み込みます
  • ドキュメントのバグのトリアージと優先順位付けを支援します。バグ情報に基づいて文書を書きます。ステータスに関するドキュメントのバグ (Bugzilla) を更新してください
  • 他人によって書かれたドキュメントについてのフィードバックを提供し、編集し、明確にし、そして校正します
  • スタンドアップ、ユーザーストーリーの作成、計画セッションなどのアジャイルチームプロセスに参加します。 ユーザーストーリー、承認基準、およびタスクを定義するのに役立ちます
  • 現在の開発者向けドキュメントのギャップを識別し、優先順位付けを提案します
  • コンテンツ戦略とロードマップに関するフィードバックを提供します
  • ユーザの成功とコンテンツ制作を容易にする MDN プラットフォームの改善を提案します
  • 作家、開発者、翻訳者のボランティアコミュニティと積極的かつ積極的に交流します
  • ソーシャルメディアやニュースレターにコンテンツを投稿する

必要な資格

  • Web 開発者向けのドキュメント、コードサンプル、およびデモの調査および作成に関連した2年以上の経験
  • JavaScript、ブラウザ API、CSS、および HTML を使ったコーディング経験
  • Git と Github の使用経験
  • バグ追跡ソフトウェア (Bugzilla など) に精通していること
  • アジャイルや SCRUM 方法論に精通していること
  • 技術仕様 (W3C や TC39 など) を読んで理解できる
  • HTML、Wiki、その他のオンラインコラボレーション技術を使用して情報を公開する能力
  • ペースの速い、絶えず変化する環境に対処する能力
  • 明確に英語で書くことができる。しかしながら英語は第一言語である必要はない
  • ガイド/チュートリアルおよび API リファレンスドキュメントを書く経験がある
  • 地理的に分散した従業員とボランティアのチームと仕事をする能力

歓迎条件

  • オープンソースコミュニティでの作業経験、MDN や Mozilla に貢献しているなら追加のボーナス。
  • 最新の Web 開発ワークフロー (Linter などの使用) の経験
  • ローカライズの経験、および英語以外の他の言語の知識
  • C++、Rust、Java などの低レベルのシステムコードの経験

MDN ライターのチームマネージャである Chris Mills に cmills@mozilla.com までメールを送ってください。あなたの履歴書、あなたがその仕事にふさわしい理由を説明したカバーレター、そしてあなたが役に立つかもしれないと思うその他の補足情報を含めてください。
