TypeError.prototype プロパティは、{{jsxref("TypeError")}} コンストラクタのプロトタイプを表します。

説明

全ての {{jsxref("TypeError")}} インスタンスは TypeError.prototypeを継承します。全てのインスタンスに対してプロパティ又はメソッドを追加するために、このプロトタイプを利用できます。

プロパティ

TypeError.prototype.constructor
インスタンスのプロトタイプを生成した関数を示します。
{{jsxref("Error.prototype.message", "TypeError.prototype.message")}}
エラーメッセージ。ECMA-262 は {{jsxref("TypeError")}} 自体が message プロパティを提供するべきであると規定してるにもかかわらず、SpiderMonkey では {{jsxref("Error.prototype.message")}} を継承します。
{{jsxref("Error.prototype.name", "TypeError.prototype.name")}}
エラー名。{{jsxref("Error")}} から継承します。
{{jsxref("Error.prototype.fileName", "TypeError.prototype.fileName")}}
エラーが発生したファイルのパス。{{jsxref("Error")}} から継承します。
{{jsxref("Error.prototype.lineNumber", "TypeError.prototype.lineNumber")}}
エラーが発生したファイル内の行番号。{{jsxref("Error")}} から継承します。
{{jsxref("Error.prototype.columnNumber", "TypeError.prototype.columnNumber")}}
エラーが発生した行内のカラム番号。{{jsxref("Error")}} から継承します。
{{jsxref("Error.prototype.stack", "TypeError.prototype.stack")}}
スタックトレース。{{jsxref("Error")}} から継承します。

メソッド

{{jsxref("TypeError")}} プロトタイプオブジェクトが自分自身のいかなるメソッドも持っていなくても、{{jsxref("TypeError")}} インスタンスはプロトタイプチェーンを通していくつかのメソッドを継承しています。
