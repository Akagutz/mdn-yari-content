패키지된 확장 기능은 파이어폭스에서 "XPI 파일"이라고 부릅니다. 이것은 ZIP파일과는 다른 확장기능입니다.

AMO로 업로드 할 때는 XPI 확장 기능을 사용할 필요가 없습니다.

개발을 하는 동안, 확장 기능은 manifest.json 파일과 필요한 다른 파일 (스크립트, 아이콘, HTML 문서 등)을 포함하는 디렉토리가 생성됩니다. AMO에 업로드하려면 이 파일을 하나의 파일로 압축해야합니다.

web-ext를 사용하는 경우 web-ext 빌드를 사용하여 확장 프로그램을 패키지화하십시오. 그렇지 않은 경우 하단의 운영 체제별 지침을 따르십시오.

Tip. ZIP 파일은 확장 기능이 들어있는 디렉토리가 아닌 확장 기능을 압축한 ZIP파일 이어야 합니다.

Windows

  1. 확장기능이 포함된 디렉토리를 여세요
  2. 모든 파일과 디렉토리를 선택 해 주세요
  3. 단축메뉴를 열어서 보내기 에 있는 압축(ZIP) 폴더를 선택하세요

Mac OS X

  1. 확장기능이 포함된 디렉토리를 여세요
  2. 모든 파일과 디렉토리를 선택 해 주세요
  3. 단축메뉴를 열어서 n 개의 아이템 압축하기를 선택하세요.

See http://www.info-zip.org/mans/zip.html.

Linux / Mac OS X Terminal

  1.  Terminal을 여세요.
  2. 아래의 명령어를 이용하여 확장 기능이 포함된 디렉토리를 여세요.
    cd path/to/my-extension/
  3. 아래의 명령어를 사용해 디렉토리의 내용을 압축하세요.
    zip -r -FS ../my-extension.zip *
