After Document Migration

After you migrated a page, please do the following:

  1. Make sure that you have added category information to your newly created article, so that it also can be found!
  2. Add the MDC:Examples:Original Document Information information block to your document, correctly listing the original author(s) and contributors. You can find the last modification date in the page footer at the bottom of the original page on www.mozilla.org.
  3. Assert the migration by adding " -- migrated toname of new page" to the entry of the migrated page and move it to the top of the Migrated list below (so nobody wastes its time by migrating it again).
  4. Add theoriginal location and thenew location to the Needs Redirect page, so we can get redirects set up at the old locations.
  5. Edit theexternal redirect of the page you migrated (by copying the url of the redirection page to the location bar, and add "?action=edit"):
    1. Change the REDIRECT statement to link to yournew page, instead of to theexternal page.
    2. Remove all category information, and replace it by [[Category:Junk]].
    3. Click on the "What links here" link, and change every page which links to theredirection page to link to yournew page instead.
