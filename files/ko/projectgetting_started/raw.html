MDC에 도움을 주는 것은 당신의 브라우저를 사용하는 것만큼 쉽습니다. 웹에서 글을 작성해서 올려본 경험이 있다면 시작하기에 충분합니다.

소개

이곳의 위키에서 글을 쓰고, 편집하고, 리뷰하는 것을 환영합니다. 협동해서 개선하는것을 염두에 두면서, 다른 사용자가 여러분이 올린 글을 수정 또는 개선 했다고 해서 실망할 필요는 없습니다. 우리는 모질라 문서가 발전하고, 협력하고 서로 도와주는 것을 항상 환영합니다.

기본

MDC 위키에 대한 간략한 가이드는 여기에 있습니다. 위키 문법을 여기서 연습해 보세요 Sandbox.

페이지들 편집하기

페이지를 편집하기 위해서는, 먼저 위키에 계정을 만들어야 합니다. User login 페이지로 가서 가입하고 폼을 전송하면 계정을 만들 수 있습니다. 일단 계정이 만들어지면, 자동으로 로그인이 되고 편집도 할 수 있습니다.

모든 페이지의 오른쪽 칼럼에는 "편집" 링크를 볼 수 있습니다. 이 링크를 클릭함으로써 페이지의 수정폼으로 가게 되고, 그곳에서 여러분이 수정한 내용의 미리보기와 저장도 할 수 있습니다. 수정한 내용을 저장한 후에는 페이지에서 즉시 볼 수 있습니다.

새 페이지 시작하기

새 페이지를 시작하기에 가장 쉬운 방법은 당신의 사용자 페이지에 링크를 생성한 후에 그 링크를 사용해서 "편집" 모드로 들어가는 것입니다.

유저 페이지로 가기 위해서 위키 상단에 푸른색 막대안에 있는 사용자 이름을 클릭하세요. 당신의 사용자 페이지가 추가된 적이 없었다면, 즉시 페이지가 수정폼으로 갈것입니다.

페이지에 링크를 추가하는 것은 페이지의 제목을 설정하는 것만큼 간단합니다. 예: [[Here is an example]]. 저장 후에는 다음처럼 링크로 변합니다. 예: Here is an example.

페이지가 존재 하지 않으면 링크는 빨간색으로 표시 됩니다. 링크가존재 하면 파란색으로 표시 됩니다. 새 페이지를 생성하려고 하는데 그 페이지 제목이 이미 존재한다면 기존의 페이지를 편집하고 개선하는 것이 더 나을지 생각해 봐야 합니다. 그렇지 않다면 새 페이지의 제목을 다른 것으로 변경하고 링크를 생성 하세요.

새 페이지에 내용을 추가하려면, 그냥 빨간색 링크를 클릭하고 글 작성을 하면 됩니다. 당신의 작업을 저장하기 위해서 수정 폼의 "저장" 버튼 사용을 잊지 마세요 (긴 글을 작성중이라면 자주 저장하는 것도 명심).

페이지 명명 가이드

새 페이지들의 생성 및 명명에 대한 더 자세한 정보는 다음을 보세요 page naming guide.

위키 마크업

오른쪽 컬럼의 "편집"을 클릭하면 페이지내에 "==" 마크와 링크 주변의 "[{{ mediawiki.external(' ]') }}" 같은 몇몇의 특수 문자 텍스트를 보게 될것입니다. 이것은 위키 시스템의 문자 마크업의 매우 단순한 형태입니다. 이 마크업 시스템의 힘은 배우기 쉽고 실수에 관대한 것입니다.

위키 마크업 자료

위키 편집시에 마크업 언어를 배우기 위해서는 wiki markup reference 참조 하세요.

쓰기, 편집, 리뷰

여기에 작성자, 에디터, 리뷰어를 위한 다소 긴 MDC 소개글이 있습니다

작성자들

MDC는 새로운 기사에 항상 흥미가 있으며, 그래서 당신이 topics에 해당하는 새로운 기사를 쓰는 것을 무엇이든 환영 합니다. 새로운 컨텐츠는 한페이지로 된 팁처럼 단순할 수도 있고 프로그래밍 가이드북이나 레퍼런스 만큼 복잡할 수도 있습니다. 새 컨텐츠의 분량은 당신이 기고하는 분량에 달려 있습니다.

모질라 프로젝트의 버그들은 Bugzilla database 영향받는 문서(impact documentation)가 dev-doc-needed 키워드를 꼬리표로 달아야만 합니다. 이것은 문서를 필요로 하는 항목들을 발견하는 하나의 중대한 방법입니다. 전형적으로, 수정사항들이 소스 코드로 이미 검사된 버그들만 자세히 작성됩니다. 필수 문서의 수정사항들의 작성이 이루어지면, dev-doc-complete 키워드는 버그에 추가되어야 합니다.

큰 프로젝트를 맡고 있다면, MDC community 에 가입하고 그곳에서 유사한 프로젝트를 시작하려고 하는 사람과 토론하는 것을 권유 합니다.

위키내에서 사용하는 스타일이나 사용법 정책에 질문이 있으면 writer's guide를 참조 하세요.

편집자들

특별히 글을 개선 하거나 스타일, 문법, 철자를 수정하는 것들을 포함해서 위키를 편집하는 것은 언제나 환영합니다, . 더 자세한 정보는 writer's guide 를 보세요.

리뷰어들

기술적인 개선이나 리뷰는 항상 환영합니다. 페이지를 읽다가 기술적인 에러(코드나 텍스트)를 발견하면, 편한 마음으로 수정하면 됩니다. 만약 기존 내용과 상반되는 변경 이라면, "talk"페이지에(오른쪽 칼럼의 "Discuss" 링크를 클릭으로 접근) 노트를 만들고 변경 이유에 대한 요점을 노트로 남겨 주세요.

작성자 가이드

우리는 선호되는 문서 작성 스타일에 대한 요약된 예제가 있습니다 writer's guide. 만약 그곳에서 다루지 않은 사용법과 스타일에 대한 질문이 있으면 Economist style guide 또는 , Chicago Manual of Style 을 참조 하세요.

우리는 가이드를 계속 확장할 것이므로, writer's guide 에서 다루지 못한 특정 질문이 있으면 MDC mailing list 또는 developer documentation lead 으로 보내 주세요.

오래된 문서 이주 하기

오래동안 진행중인 MDC의 하위 프로젝트중 하나는 mozilla.org 웹사이트의 개발자 문서를 MDC 위키로 이주 하는 것입니다. 실행 절차는 대체로 간단 합니다:

  1. existing content 목록에서 이주하길 원하는 문서를 고릅니다.
  2. page naming guide를 참조해서 가져온 문서의 위키내에서의 제목을 정합니다.
  3. 페이지들을 생성하고 내용을 가져온다음, wiki markup reference를 가이드로 해서 마크업 합니다.
  4. existing content 목록에서 해당 아이템을 삭제 합니다.
  5. needs redirect 목록에 해당 아이템을 추가 합니다.

만약 이 작업을 하는데 포괄적인 방법을 좋아한다면, 이렇게 할 수도 있습니다:

  1. our search system를 사용해서 위키의 원본 문서의 링크들을 검색한 후에, 그 링크들을 새 위키 페이지를 가리키도록 변경 합니다.
  2. external redirects로 원본 문서를 체크하고, [[Category:Junk]] 추가로 "Junk" 분류로 넣어져서 삭제된 것에 대한 리다이렉트들을 표시 합니다.

MDC 지역화

다른 언어로 된 MDC 위키의 문서 번역을 돕는데 흥미가 있으면, 간단히 관련된 언어의 위키를 방문해서 페이지 생성 또는 수정을 해주세요.

기존의 지역화 프로젝트들

현재의 MDC는 다음 언어들로 된 위키들이 있습니다: Català, 中文(简体), Deutsch, English, Français, Italiano, 日本語, Nederlands, Polski, Português, Español, 한국어, 中文(正體), Русский.

새 지역화 프로젝트를 시작하는 방법

우리 위키에 없는 언어로 된 새로운 지역화 프로젝트를 시작하고 싶다면, localization projects 페이지를 읽고 그곳의 지시를 따라 주세요.

"edit wars" 해결하기 또는 기타 충돌들

무슨 이유든 "edit war" 페이지가 발생된다면, 페이지내에 두명이상의 사용자가 계속적이고 지속적으로 만들고 변경했다는 것이며, 충돌을 풀려면 먼저 참여한 사용자가 충돌을 해결하려고 노력해야 합니다.

해결책이 없으면, 그것을 MDC mailing list 또는 (최후의 수단으로써) developer documentation lead로 보내서 관심을 갖도록 (원칙적으로) 합니다. 다른 기술 전문가들, 저술가들, 편집자들의 조언을 구하면, 그들의 전문적 기술이 문제를 해결하는데 도움을 줄 것입니다.

페이지 삭제를 요청하기

실수로 페이지를 만들거나 우연히 삭제되어야 할 페이지를 발견하면 [[Category:Junk]]를 페이지 하단에 추가함으로서 "Junk" 카테고리로 추가하세요.

Junk category는 위키 관리자들의 의해서 주기적으로 체크되어서 적절한 시점에 평가되고 삭제될 것입니다.

Note: "Junk"로 표시한 페이지의 다른 내용이나 카테고리들을 삭제 하지 말아 주세요.
