Location.reload() 메서드는 새로고침 버튼처럼 현재 리소스를 다시 불러옵니다.

{{Non-standard_Inline}} 일부 브라우저에서는 선택적으로 {{domxref("Boolean")}} 매개변수를 제공할 수 있습니다. (기본값 false) true를 지정한 경우 페이지를 항상 서버에서 불러오고, 브라우저의 HTTP 캐시를 무시합니다.

새로고침은 SECURITY_ERROR {{domxref("DOMException")}}과 함께 거부당할 수 있습니다. 이는 location.reload()를 호출한 스크립트의 {{glossary("origin", "출처")}}와 {{domxref("Location")}} 객체를 소유한 문서의 출처가 다를 때 발생합니다. 더 자세한 정보는 동일 출처 정책 문서를 참고하세요.

구문

location.reload(forcedReload);

매개변수

forcedReload {{optional_inline}} {{Non-standard_Inline}}
페이지를 항상 서버로부터 불러와야 할지 나타내는 {{domxref("Boolean")}}. 값을 지정하지 않았거나 false인 경우 브라우저가 HTTP 캐시를 사용해 페이지를 새로 고칠 수 있습니다.

예제

// 브라우저 캐시 없이 새로고침
window.location.reload(true);
