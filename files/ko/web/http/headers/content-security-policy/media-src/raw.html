HTTP {{HTTPHeader("Content-Security-Policy")}} (CSP) media-src 구문은{{HTMLElement("audio")}} 와 {{HTMLElement("video")}} 엘리먼트를 사용하여 미디어를 로드하기 위해서 유효한 출처를 지정하기 위해서 사용합니다.

CSP version 1
Directive type {{Glossary("Fetch directive")}}
{{CSP("default-src")}} fallback Yes. If this directive is absent, the user agent will look for the default-src directive.

Syntax

media-src 정책은 하나 또는 그 이상의 출처를 허용할 수 있습니다:

Content-Security-Policy: media-src <source>;
Content-Security-Policy: media-src <source> <source>;

Sources

{{page("Web/HTTP/Headers/Content-Security-Policy/default-src", "Sources")}}

Examples

Violation cases

아래가 주어진 CSP Header:

Content-Security-Policy: media-src https://example.com/

아래의 {{HTMLElement("video")}}, {{HTMLElement("video")}} and {{HTMLElement("track")}} 태그는 차단되고 리소스를 불러오지 않습니다.:

<audio src="https://not-example.com/audio"></audio>

<video src="https://not-example.com/video"> 
  <track kind="subtitles" src="https://not-example.com/subtitles"> 
</video>
