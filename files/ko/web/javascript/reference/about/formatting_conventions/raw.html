JavaScript 응용프로그램은 많은 운영체제에서 작동합니다. 따라서 이 책의 정보는 모든 버전에 적용합니다. 파일과 디렉토리 경로는 Windows 포맷(\로 디렉토리명을 구분하는)으로 제공합니다. Unix에서, 디렉토리 경로는 디렉토리를 구분하는데 \ 대신 빗금(/)을 쓰는 것만 빼고 같습니다.

이 책은 다음 형식의 uniform resource locators (URLs)를 씁니다.

http://server.domain/path/file.html

이 URL에서

  • server는 research1이나 www 같은 응용프로그램이 작동하는 서버명을 나타냅니다.
  • domain은 netscape.com이나 uiuc.edu 같은 인터넷 도메인명을 나타냅니다.
  • path는 서버 위의 디렉토리 구조를 나타냅니다.
  • file.html은 개별 파일명을 나타냅니다.

보통, URL에서 이탤릭체로 쓴 항목은 개체 틀(placeholder)이고 보통 고정폭(monospace) 글꼴로 쓴 항목은 상수(literal)입니다.

이 책은 다음 글꼴 규약(convention)을 씁니다.

  • monospace font(고정폭 글꼴)은 견본 코드와 코드 나열, API, 언어 요소(메소드명과 속성명 같은), 파일명, 경로명, 디렉토리명, HTML 택, 화면에 타이핑해야 할 텍스트에 씁니다. (고정폭 이탤릭체 글꼴은 코드에 끼워넣는 개체 틀에 씁니다.)
  • Italic type(이탤릭체)는 책 제목, 강조(emphasis), 변수, 개체 틀, 상수 뜻으로 쓰는 낱말에 씁니다.
  • Boldface type(볼드체)는 용어집 용어에 씁니다.
