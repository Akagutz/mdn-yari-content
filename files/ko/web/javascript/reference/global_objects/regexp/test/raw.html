{{JSRef}}

정규표현식의 test() 메소드는 대상 문자열 속에 일치하는 문자열이 포함되어 있는지 검사하고 true 또는 false를 반환한다. 

문법

regexObj.test(str)

인자

str
정규표현식 검사를 수행할 대상 문자열.

반환값

Boolean. true or false.

설명

문자열에 일치하는 패턴이 있는지 알고 싶을 때는 {{jsxref("RegExp.prototype.test()", "test()")}} 메소드를 사용한다. (참고로 {{jsxref("String.prototype.search()")}} 메소드는 일치할 때 index, 그렇지 않으면 -1 을 반환하지만 {{jsxref("RegExp.prototype.test()", "test()")}} 메소드는 boolean을 반환하는 것이 다른 점임);   for more information (but slower execution) use the {{jsxref("RegExp.prototype.exec()", "exec()")}} method (similar to the {{jsxref("String.prototype.match()")}} method). As with {{jsxref("RegExp.prototype.exec()", "exec()")}} (or in combination with it), test() called multiple times on the same global regular expression instance will advance past the previous match.

Examples

Using test()

Simple example that tests if "hello" is contained at the very beginning of a string, returning a boolean result.

var str = "hello world!";
var result = /^hello/.test(str);
console.log(result); // true

The following example logs a message which depends on the success of the test:

function testinput(re, str){
  var midstring;
  if (re.test(str)) {
    midstring = ' contains ';
  } else {
    midstring = ' does not contain ';
  }
  console.log(str + midstring + re.source);
}

Gecko-specific notes

Prior to Gecko 8.0 {{geckoRelease("8.0")}}, test() was implemented incorrectly; when it was called with no parameters, it would match against the value of the previous input (RegExp.input property) instead of against the string "undefined". This is fixed; now /undefined/.test() correctly results in true, instead of an error.
