Apesar de nos esforçarmos para fazer a documentação no MDN Web Docs a mais clara e legível possível, é inevitável que certas convenções ou escolhas de design sejam confusas para novatos ao conteúdo, ou para pessoas ainda não familiarizadas com mudanças recentes de design. Este guia é para os leitores que querem responder essas perguntas.

Ícones de status da especificação

Em diversos lugares no MDN, como páginas iniciais e índices, utilizamos ícones para indicar o status da especificação de um elemento, propriedade, método, ou até mesmo de uma API inteira. Ao passar o seu mouse sobre estes ícones, será exibida uma dica que explica o que o ícone significa. Se não aparecer, você pode consultar esta seção para ajudar a entendê-los.

{{FontAwesomeIcon("icon-warning-sign")}} Não-padrão; o item não faz parte de qualquer especificação de padrões.
{{FontAwesomeIcon("icon-thumbs-down-alt")}} Depreciado; o item não é mais recomendado para uso, mesmo que as especificações ainda permitam que seja utilizado por questões de legado.
{{FontAwesomeIcon("icon-trash")}} Obsoleto; o item foi removido das especificações e não deve ser utilizado. Você deve tentar remover qualquer uso existente de seu código.
{{FontAwesomeIcon("icon-beaker")}} Experimental; o item é atualmente considerado para especificação, ou pode ser parte de um rascunho antecipado de uma especificação, e possivelmente sofrerá mudanças antes de ser aceito. Você não deve utilizá-lo em código de produção sem ter a certeza de que seu código ainda funcionará mesmo se esta funcionalidade não for executada como esperada.
