A Rede de Desenvolvedores da Mozilla (MDN) tem tarefas para atender qualquer nível de conhecimento técnico. Você não precisa ser um programador ou saber muito sobre tecnologia. Há uma gama de tarefas que precisam ser feitas, a partir do simples (prova de leitura e correção de erros de digitação) para o complexo (escrevendo documentação da API).

MDN tem uma interface simples baseada na web que será familiar a qualquer um que tenha usado um processador de texto. Por favor, ajude-nos! O sucesso da MDN é construído no espírito de melhoria cooperativa por uma comunidade de Mozillians.

  • Conteúdo é mais importante do que estilo, gramática e ortografia. Se o seu inglês não é perfeito - não se preocupe com isso! Se você não memorizou o guia de estilos - não se preocupe com isso! Se você cometeu um erro de digitação - não se preocupe com isso! Erros são fáceis de corrigir e mudanças podem ser rapidamente e facilmente revertidas.
  • Peça ajuda e converse com seus novos amigos na comunidade MDN. A comunidade (o pessoal remunerado e os voluntários) são amigáveis. Nós amamos (AMAMOS!) novos contribuidores! Se você não receber uma resposta rápida para seu email ou IRC questão, lembre que há pessoas trabalhando em diferentes fusos horários em diferentes projetos. Se seinta livre para perguntar mais de uma vez ou para mais de uma pessoa. Você não será ignorado. Entre em contato com Eric Shepherd, o líder da MDN, se você não estiver conseguindo a ajuda que precisa. Nada é mais importante para nós do que ajudar contribuidores.
  • Qualquer um pode mudar qualquer coisa. Ninguem deve ficar chateado se você mudar seu conteudo, e você não deve ficar chateado se alguém mudar suas coisas (embora você deva se sentir livre para perguntar sobre mudanças). Mudanças não são críticas - elas são (quase sempre) mudanças construtivas.
  • Nós temos drivers para tópicos de áreas especificas. Se você está trabalhando em alguma área particular, nós podemos ajudar.

Guia para começar

Log in

MDN usa Persona para autenticação. Crie uma conta Persona e depois faça login no MDN.

Pratique

Pratique suas habilidades na Sandbox, uma area da wiki que para brincar, aprender e experimentar.

Edite uma página

  1. Clique no botão "Edit" no topo de qualquer página
  2. Faça suas alterações. Clique "PREVIEW CHANGES" para ver como sua edição ficará.
  3. Clique "SAVE AND KEEP EDITING" para salvar suas alterações sem fechar a interface de edição. Clique "SAVE CHANGES" para salvar a página e fechar a interface de edição. Clique "DISCARD CHANGES" para reverter a versão original da página.

Você pode não ser capaz de salvar suas mudanças e irá aparecer um erro de "Permission Denied" se seu browser estiver configurado para não enviar informações de referência. Se você não conseguir salvar suas alterações, no Firefox, tenha certeza que a preferência{{pref("network.http.sendRefererHeader")}}esttá configurada com o valor padrão(at the moment it's "2").

Criando uma nova página

Se você não sabe onde por uma nova página, não se preocupe! Coloque-a em qualquer lugar que alguém mais tarde a moverá para o seu lugar certo. Ou então coloque o conteúdo em uma página existente que estiver em contexto. E você não precisa se preocupar em deixar tudo perfeito. Nós temos muitos gnomos que ficarão felizes em deixar o seu texto bem formatado e atraente.

Linkando uma página

  1. Escreva o nome da nova página em qualquer lugar da primeira página.
  2. Selecione o nome e clique no ícone Inserir/Editar Link () na barra de edição. A janela de Editar Link abrirá com o nome realçado no campo "Link Text".
  3. "/en-US/docs/", por padrão, será automaticamente prefixado no campo da URL. Coloque o nome da página no final de "/en-US/docs/". (O nome da página não precisa ser o mesmo do link.)
  4. Clique OK para criar e inserir o link. Pronto!

Se a página que você linkou não existir, o link ficará formatado em vermelho. Mas se a página existe, o link estará em azul. E se você quiser criar uma nova página, mas o título em que você pensou já está em uso, analise se não vale mais a pena ajudar a editar e melhorar essa página que já existe. Caso contrário, pense em um título diferente para a sua nova página e crie um link para ela. Consulte o guia para nomear uma página.

Para adicionar conteudo a sua nova pagina, clique no link vermelho que você acabou de criar (após salvar e fechar o editor) e comece a escrever. Clique em "SAVE AND KEEP EDITING" ao longo do seu trabalho para salvar sem sair da página de edição. Quando você terminar, clique em "SAVE CHANGES".

New page without link

To create a new page without linking from another page, enter a unique page name in the URL field of your browser. For example, if you enter:

https://developer.mozilla.org/en-US/docs/FooBar

... MDN will create a new page with the title "FooBar" and open the editor so you can add content to the new page.

Wiki markup

Kuma, the Mozilla-developed software the Mozilla Developer Network uses, uses HTML as its markup language for the documentation. If you wish to see or edit the HTML content, you can do so by clicking the "Source" button in the toolbar while in edit mode. Click it again to return to the WYSIWYG editor.

Logging documentation bugs

You can report complex problems by filing a documentation bug. Use these field values:

product
Mozilla Developer Network
component
Documentation requests
URL
The page where you found the problem
Description
As much as you know or have time to describe about the problem and where to find correct information. This can include people ("talk to so-and-so") as well as Web links.

Migrating old documentation

One longer ongoing subproject at the MDN is to migrate all currently existing developer documentation from the mozilla.org web site into the MDN wiki. The procedure is fairly simple:

  1. Pick a document you would like to migrate from the existing content list.
  2. Refer to the page naming guide and decide what the page or pages for that document should be called in the wiki.
  3. Create those pages and migrate the content, marking it up as best you can using the wiki markup reference as a guide.
  4. Delete the item from the existing content list.
  5. Add the item to the needs redirect list.

If you are keen on doing a comprehensive job on this, you will also:

  1. Use our search system to search for any links to the original document in the wiki, then change those links so they point at the new wiki page.
  2. Check for external redirects to the original document, and flag those redirects for deletion by adding the "Junk" tag to them.

Localizing MDN

If you are interested in helping translate the documentation in any of the MDN wikis to another language, simply visit the relevant language's wiki and create or edit pages. See Localization projects for information about current projects and how to start a new project.

Resolving "edit wars" or other conflicts

If for whatever reason you end up in an "edit war" on a page, where two or more people are continually and repeatedly making and reverting changes, please try to resolve the conflict with the people involved first.

Should no resolution be possible, please bring it to the attention of either (ideally) the MDN mailing list or (as a last resort) the developer documentation lead. Other technical experts, writers, and editors will be consulted as required, and their expertise will be used to resolve the problem.

Requesting page deletions

Should you create a page by accident or stumble across a page you think should be deleted, mark it as such by adding the "Junk" tag to it.

Items tagged as Junk are checked periodically by wiki administrators and the pages it contains are evaluated and deleted when appropriate.

Note: Please do not delete other content or categories from the pages you flag as "Junk".
