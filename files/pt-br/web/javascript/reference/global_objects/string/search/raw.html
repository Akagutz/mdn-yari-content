{{JSRef}}

O método search() executa uma busca de uma expressão regular neste objeto  {{jsxref("String")}}.

Sintaxe

str.search(regexp)

Parâmetros

regexp
Um objeto contendo uma expressão regular. Se um objeto obj for passado como argumento e não for do tipo RegExp, ele é implicitamente convertido para um objeto {{jsxref("RegExp")}} através da instrução new RegExp(obj).

Valor retornado

O índice no string do primeiro trecho que satisfez a expressão regular; se não encontrado, -1.

Descrição

Quando quiser verificar um padrão no texto e inclusive seu índice na string utilize search() (se apenas quiser verificar então use o método similar {{jsxref("RegExp.prototype.test()", "test()")}} no prototype RegExp, que retorna um booleano); para mais informações (porém executado mais lentamente) use {{jsxref("String.prototype.match()", "match()")}} (similar ao método {{jsxref("RegExp.prototype.exec()", "exec()")}}).

Exemplos

Usando search()

O seguinte exemplo efetua o log de uma mensagem que depende do sucesso do teste.

function testinput(re, str) {
  var midstring;
  if (str.search(re) != -1) {
    midstring = ' contem ';
  } else {
    midstring = ' nao contem ';
  }
  console.log(str + midstring + re);
}

Notas específicas do motor Gecko

  • Antes de {{Gecko("8.0")}}, search() foi implementado incorretamente; quando era invocado sem parâmetros ou com {{jsxref("undefined")}}, ele buscava pelo string 'undefined', ao invés de buscar pelo string vazio. Isto foi corrigido; Agora 'a'.search() e 'a'.search(undefined) corretamente retornam 0.
  • A partir do Gecko 39 {{geckoRelease(39)}}, o argumento não-padrão flags está defasado (deprecated) e dispara uma console warning ({{bug(1142351)}}).
  • A partir do Gecko 47 {{geckoRelease(47)}}, o argumento não-padrão flags não é mais suportado em builds non-release e em breve será removido inteiramente ({{bug(1245801)}}).
  • A partir do Gecko 49 {{geckoRelease(49)}}, o argumento não-padrão flags não é mais suportado ({{bug(1108382)}}).
