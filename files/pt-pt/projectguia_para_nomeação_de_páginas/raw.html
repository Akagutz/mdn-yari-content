A nomeação de páginas no wiki do MDC é relativamente direta, mas é importante que você faça corretamente a fim de obter todas as vantagens da extensão breadcrumbs que é uma extensão personalizada adicionada por nós.

Por favor consulte também o Guia do escritor ao escolher um nome de página.

Convenções gerais

Capitalização

Nomes de páginas devem seguir uma regra de capitalização de forma que apenas sua primeira palavra seja capitulada:

  • Correto: "Um novo método para criar roolovers Javascript"
  • Incorreto: "Um Novo Método Para Criar Roolovers Javascript"
Por enquanto estas regras de capitalização se aplicam somente à páginas novas. Há um grande número de páginas em nosso Wiki que não estão conformes com estas regras, mas por favor não corrija-as por enquanto. Nós iremos corrigir isto futuramente. Obrigada. -- dria 09:20, 14 de Março de 2006 (PST)

Artigos de uma única página

Conteúdo com um único título

Se você está adicionando um artigo ou outro conteúdo que somente requer uma página simples no wiki, simplesmente use o nome deste artigo como o nome da página.

Por exemplo, se o artigo de uma única página for chamado "Um novo método para roolovers javascript" a sua página deve se chamar, simplesmente Um novo método para roolovers javascript. Assim, o título de seu conteúdo será único, esta é a forma mais fácil e deve ser usada, preferencialmente.

Conteúdo com mais de um título

Sempre que possível, você deve tentar criar conteúdos com títulos únicos, de forma a simplificar a nomeação das páginas o máximo possível. Se isto não for possível, prefixe o título do artigo com seu respectivo tópico, seguido de dois pontos e sem espaços.

Por exemplo, se eu precisar adicionar uma página chamada "Otimização", simplesmente chamá-la de "Otimização" possivelmente criará um conflito, já que "Otimização" pode ser aplicado a uma larga variedade de tópicos no Wiki. Se meu artigo "Otimização" for especialmente sobre CSS, eu devo aumentar o meu título, de forma que ele se torne único (ex: CSS:Otimização).

Se houver duas ou mais páginas com o mesmo título, uma Página de "desambiguação" deve ser criada.

Esta é a melhor prática para que nunca haja artigos com o mesmo título.

Artigos de múltiplas páginas

Se o conteúdo que você está adicionando requer múltiplas páginas, use o seguinte método de nomeação. Este método é usado vantajosamente pela extensão breadcrumbs.

  • Aprendendo JavaScript - Página com a tabela de conteúdo principal
  • Aprendendo JavaScript:Introdução
  • Aprendendo JavaScript:Visão de especialistas
  • Aprendendo JavaScript:Programação mais avançada
  • Aprendendo JavaScript:Próximos passos

Notar o prefixo "Aprendendo JavaScript" no título de cada página. De forma a fazer a extensão breadcrumbs funcionar, cada página que estiver depois dos "dois pontos" deve existir separadamente.

Se você desejar usar a extensão breadcrumbs para hierarquia de navegação em suas páginas, você deve usar a delimitação por "dois pontos" e a nomeação descrita aqui.
