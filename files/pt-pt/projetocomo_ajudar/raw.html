Introdução
Você quer ajudar a melhorar ainda mais o MDC? Veja como começar.

O Mozilla Developer Network (MDN) apoia o crescimento e desenvolvimento do Firefox e da web, fornecendo informações abrangentes, precisas e atualizadas sobre o Firefox, Mozilla e tecnologias de desenvolvimento web. O Mozilla Documentation Center (MDC) é a seção de documentação do MDN.

Se você está procurando ajuda técnica, tente navegar ou pesquisar o site. Questões técnicas devem ser dirigidas a um dos fóruns de desenvolvimento da Mozilla.

Perguntas dos usuários podem ser feitas em um dos fóruns de suporte ao usuário final.

Encontrei uma informação errada ou desatualizada. O que devo fazer?

Idealmente, seria ótimo se você pudesse apenas corrigi-lo. Afinal, o MDC é um wiki. Para erros de digitação e outras coisas menores, é realmente mais rápido para consertá-lo do que contar a alguém sobre isso.

Mas isso nem sempre é realista. Talvez você veio para MDC para encontrar rapidamente alguma informação, enquanto fazia alguma outra tarefa, e você não quer interromper o seu rendimento. Ou talvez você possa dizer que algo não está certo, mas você não conhece exatamente toda a informação correta. Existem algumas maneiras que você pode sinalizar um problema com a esperança de ser corrigido por outra pessoa, incluindo o Bugzilla e marcação.

Bugzilla

Se você não sabe o que é o Bugzilla, você pode ignorar esta seção. Se você é do tipo de pessoa que vive e respira no Bugzilla, você pode registrar um erro de documentação. Use esses valores de campo:

classification
Outro
product
Documentação do Desenvolvedor
URL
A página em que você encontrou o problema
Description
O máximo do que você sabe ou tem tempo para descrever sobre o problema e onde encontrar a informação correta. Isso pode incluir pessoas ("falar com fulano de tal"), bem como links da web.

Taggeando

Se você não é um especialista do Bugzilla, você pode usar tags dentro do MDC para sinalizar problemas. Aqui estão algumas das marcas que usamos, e seus significados:

  • NeedsMarkupWork: a formatação não é padrão ou não consistente com outras páginas.
  • NeedsExample: necessita de um ou mais exemplos de código ilustrativo do item documentado.
  • NeedsContent: o item está incompleto e precisa ser preenchido.
  • NeedsJSVersion: precisa de informações sobre a versão de JavaScript e EcmaScript de onde este item apareceu.
  • NeedsBrowserCompatibility: precisa de uma tabela compatível do navegador ou precisa completar a tabela.
  • MakeBrowserAgnostic: o artigo é escrito com foco no Gecko, quando na verdade é sobre uma função padrão ou recurso, que deve ser reescrito para ser genérico.

Para adicionar tags

  1. Faça o Login no MDN.
  2. Clique Editar para entrar no modo de edição.
  3. Mova o cursor até o final da página.
  4. Na caixa de tags, escreva a tag que voce deseja adicionar. Os tags podem conter espaços.
  5. Pressione Enter para confirmar a tag.
  6. Clique em Salvar Alterações quando acabar.

Para remover tags

  1. Faça o Login no MDN.
  2. Clique Editar para entrar no modo de edição.
  3. Mova o cursor até o final da página.
  4. Clique no X ao lado da tag que deseja deletar.
  5. Clique em Salvar Alterações quando acabar.

Eu quero participar!

Se você gostaria de ajudar a tornar o conteúdo do MDC melhor, existem várias maneiras que você pode contribuir. Você pode escrever novos conteúdos, editar e copiar o material existente correto, ou traduzir o material em novas línguas. Confira a Introdução para mais detalhes.
