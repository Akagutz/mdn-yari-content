Начинаем!
Вы хотите помочь сделать MDC ещё лучше? Читайте, с чего начать.

Mozilla Developer Network (MDN) (Сеть разработчиков Mozilla) поддерживает рост и развитие Firefox и веб, предоставляя полную, достоверную и актуальную информацию о Firefox, Mozilla и технологиях веб-разработки. MDN Documentation Center (Центр документации MDN - MDC) является разделом MDN для публикации документации.

Если Вам нужна техническая поддержка , используйте просмотр (browsing) или поиск по сайту (searching the site). Технические вопросы следует направлять в один из форумов по разработке Mozilla (Mozilla development forums).

Пользователи могут задавать вопросы на форумах поддержки конечных пользователей (end-user support forums).

Я нашел ошибку или устаревшую информацию. Что мне следует сделать?

В идеале, было бы здорово, если бы вы просто исправили это. MDC - это вики, в конце концов. Для опечаток и других мелочей быстрее исправить, чем сказать кому-то об этом.

Но это не всегда реально. Может быть, вы пришли в MDC, чтобы быстро найти некоторую информацию, выполняя какую-то другую задачу, и вы не хотите прерывать поток. Или вы можете сказать, что что-то не в порядке, но вы лично не можете исправить это.  Вы можете сообщить о проблеме, используя систему Bugzilla и тэги, чтобы кто-то другой решил ее.

Багзилла

Если вы не знаете, что такое "багзилла", просто смело читайте дальше . Если же вам все-таки интересно, что же такое Bugzilla, вы можете прочесть file a documentation bug. Используйте эти значения:

продукт
Mozilla Developer Network (MDN, Сеть Разработчиков Mozilla)
компоненты
Документация
URL
Страница, где вы нашли проблему
Описание
As much as you know or have time to describe about the problem and where to find correct information. This can include people ("talk to so-and-so") as well as Web links.

Расстановка тэгов

If you're not a Bugzilla maven, you can use tags within MDC to flag problems. Here are some of the tags we use, and their meanings:

  • NeedsMarkupWork: the formatting is not standard or not consistent with other pages.
  • NeedsExample: needs one or more illustrative code examples of the item documented.
  • NeedsContent: the item is incomplete and needs to be filled out.
  • NeedsJSVersion: needs information about the version of JavaScript and EcmaScript this item first appears in.
  • NeedsBrowserCompatibility: needs a browser compatibility table or needs the table filled out.
  • MakeBrowserAgnostic: the article is written with a focus on Gecko, when it is actually about a standard function or feature, which should be rewritten to be generic.

Как добавить тэг:

  1. Зайдите в MDN.
  2. Нажмите "изменить", чтобы перейти в режим изменения.
  3. Пролистайте страницу вниз.
  4. Внизу введите в поле "добавить тэг", тот тэг, который хотите. Тэг может содержать пробелы
  5. Нажмите "Enter" для применения изменений.
  6. Нажмите кнопку "Сохранить", чтобы сохранить данные.

Как удалить тэг:

  1. Зайдите в MDN.
  2. Нажмите "изменить", чтобы перейти в режим изменения.
  3. Пролистайте страницу вниз.
  4. Нажмите крестик около того тэга, который ранее добавили.
  5. Нажмите кнопку "Сохранить", чтобы сохранить данные.
