Описание

Значение null является литералом (а не свойством глобального объекта, как {{jsxref("Global_Objects/undefined", "undefined")}}). В API null часто присутствует в местах, где ожидается объект, но подходящего объекта нет. При проверке на null или {{jsxref("Global_Objects/undefined", "undefined")}} помните о {{jsxref("Operators/Comparison_Operators", "различии между операторами равенства (==) и идентичности (===)", "", 1)}} (с первым выполняется преобразование типов).

// переменная foo не существует, она не была определена и никогда не инициализировалась:
> foo
'ReferenceError: foo is not defined'

// переменная foo сейчас существует, но она не имеет ни типа, ни значения:
> var foo = null; foo
'null'

Отличия между null и undefined

typeof null        // object (баг в ECMAScript, должно быть null)
typeof undefined   // undefined
null === undefined // false
null  == undefined // true
