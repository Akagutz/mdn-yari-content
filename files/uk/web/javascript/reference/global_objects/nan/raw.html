Опис

NaN є властивістю глобального об'єкта.

Початковим значенням NaN є "не число" — так само, як і значення {{jsxref("Number.NaN")}}. У сучасних переглядачах NaN — це властивість, недоступна для запису та налаштування. Навіть якщо у вашому випадку це не так, уникайте його перевизначення.

У програмах NaN застосовується досить рідко. Це значення повертається в результаті помилок у функціях {{jsxref("Math")}} (Math.sqrt(-1)), або коли виникає помилка при розборі числа (parseInt("blabla")).

Первірка на NaN

NaN не дорівнює (через ==, !=, === та !==) будь-якому значенню, в тому числі іншому значенню NaN. Використовуйте {{jsxref("Number.isNaN()")}} або {{jsxref("Global_Objects/isNaN", "isNaN()")}}, щоб якнайточніше визначити, чи значення є NaN. Або ж виконуйте порівняння аргументу із самим собою: тільки NaN не дорівнює самому собі.

NaN === NaN;        // false
Number.NaN === NaN; // false
isNaN(NaN);         // true
isNaN(Number.NaN);  // true

function valueIsNaN(v) { return v !== v; }
valueIsNaN(1);          // false
valueIsNaN(NaN);        // true
valueIsNaN(Number.NaN); // true

Однак, пам'ятайте про відмінність між isNaN() та Number.isNaN(): перший поверне true, якщо аргумент є NaN, або якщо він стане NaN після приведення до чисельного типу, в той час як другий поверне true лише у випадку, якщо аргумент на даний момент є NaN:

isNaN('привіт'); // повертає 'true'.
Number.isNaN('привіт'); // повертає 'false'.
