当currentTime更新时会触发timeupdate事件。

这个事件的触发频率由系统决定，但是会保证每秒触发4-66次（前提是每次事件处理不会超过250ms）。鼓励用户代理根据系统的负载和处理事件的平均成本来改变事件的频率，保证UI更新不会影响视频的解码。
