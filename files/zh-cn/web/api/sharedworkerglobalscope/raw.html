{{APIRef("Web Workers API")}}

SharedWorkerGlobalScope对象 ({{domxref("SharedWorker")}}全局作用域)可以通过 {{domxref("window.self","self")}} 关键字来获取. 某些被列举在 JavaScript Reference中可用的全局方法、命名空间对象和构造方法通常不能被worker的全局作用域所使用但是却依赖于它。查看workers的完整functions列表.

属性

这个接口继承自 {{domxref("WorkerGlobalScope")}}和父类的属性，因此实现了{{domxref("WindowTimers")}}、 {{domxref("WindowBase64")}}和 {{domxref("WindowEventHandlers")}}的属性。

{{domxref("SharedWorkerGlobalScope.name")}} {{readOnlyinline}}
name属性为当SharedWorkerGlobalScope创建时， {{domxref("SharedWorker")}} (可选择行的) 赋予的。 {{domxref("SharedWorker.SharedWorker", "SharedWorker()")}} 构造函数可以获取name值的引用并传递给 {{domxref("SharedWorkerGlobalScope")}}.
{{domxref("SharedWorkerGlobalScope.applicationCache")}} {{readOnlyinline}}
applicationCache属性为worker返回{{domxref("ApplicationCache")}}对象(查看 Using the application cache)。

属性继承自WorkerGlobalScope

{{domxref("WorkerGlobalScope.self")}}
返回一个DedicatedWorkerGlobalScope对象的引用。
{{domxref("WorkerGlobalScope.console")}} {{readOnlyinline}}
返回与worker相关的{{domxref("Console")}} 。
{{domxref("WorkerGlobalScope.location")}} {{readOnlyinline}}
返回与worker相关的{{domxref("WorkerLocation")}}。WorkerLocation是一个特殊的location对象, 通常是浏览器作用域中 {{domxref("Location")}} 的一个子集, 但是适配于workers。
{{domxref("WorkerGlobalScope.navigator")}} {{readOnlyinline}}
返回与worker相关的{{domxref("WorkerNavigator")}}。WorkerNavigator是一个特殊的navigator对象, 通常是浏览器作用域中 {{domxref("Navigator")}} 的一个子集, 但是适配于workers。
{{domxref("WorkerGlobalScope.performance")}} {{readOnlyinline}} {{Non-standard_inline}}
返回Returns the {{domxref("Performance")}} object associated with the worker, which is a regular performance object, but with a subset of its properties and methods available.

事件处理

这个接口继承自 {{domxref("WorkerGlobalScope")}}和父类的事件处理，因此实现了{{domxref("WindowTimers")}}、 {{domxref("WindowBase64")}}和 {{domxref("WindowEventHandlers")}}的事件处理。

{{domxref("SharedWorkerGlobalScope.onconnect")}}
Is an {{domxref("EventHandler")}} representing the code to be called when the {{event("connect")}} event is raised — that is, when a {{domxref("MessagePort")}} connection is opened between the associated {{domxref("SharedWorker")}} and the main thread.

方法

这个接口继承自 {{domxref("WorkerGlobalScope")}}和父类的方法，因此实现了{{domxref("WindowTimers")}}、 {{domxref("WindowBase64")}}和 {{domxref("WindowEventHandlers")}}的方法。

继承自WorkerGlobalScope

{{domxref("WorkerGlobalScope.close()")}}
记录WorkerGlobalScope的事件循环队列中的任务， effectively closing this particular scope.
{{domxref("WorkerGlobalScope.dump()")}} {{non-standard_inline}}
允许输出信息——比如在终端。这个类似于Firefox的{{domxref("window.dump")}}。
{{domxref("WorkerGlobalScope.importScripts()")}}
导入一个或多个脚本到worker的作用域内。可以用逗号来声明足够多的脚本。例如：importScripts('foo.js', 'bar.js');

实现自其它地方

{{domxref("WindowBase64.atob()")}}
将使用base-64编码的字符串解码。
{{domxref("WindowBase64.btoa()")}}
将二进制字符串编码成base-64的ASCII码字符串。
{{domxref("WindowTimers.clearInterval()")}}
取消重复执行字符集，其使用了{{domxref("WindowTimers.setInterval()")}}。
{{domxref("WindowTimers.clearTimeout()")}}
取消重复执行字符集，其使用了 {{domxref("WindowTimers.setTimeout()")}}.
{{domxref("WindowTimers.setInterval()")}}
每x毫秒安排函数执行。
{{domxref("WindowTimers.setTimeout()")}}
为函数执行设置延迟时间。
