TextEncoder 将输入的代码流指向一个字节流并输出. 现提供一个可扩展的非原生库, 参见 StringView – a C-like representation of strings based on typed arrays.

Note: Firefox, Chrome and Opera 已经支持utf-8以外的编码格式 (例如:  utf-16, iso-8859-2, koi8, cp1261, and gbk). 从Firefox 48 (ticket), Chrome 54 (ticket) 和 Opera 41开始, 为了符合规范，utf-8以外的编码格式都不可用，在任何情况下，将编码类型传递给构造函数将被忽略，并且将创建一个utf-8 TextEncoder (Note: TextDecoder 仍然允许其他解码类型). 

There is a polyfill implementation to support all the legacy encoding in github.

例子

const encoder = new TextEncoder()
const view = encoder.encode('€')
console.log(view); // Uint8Array(3) [226, 130, 172]

属性

TextEncoder 接口不会继承任何属性。

{{domxref("TextEncoder.encoding")}} {{readonlyInline}}
该属性返回一个包含编码器名称的 {{domxref("DOMString")}} , 用来描述TextEncoder将会使用的方法的字符串。

构造

{{domxref("TextEncoder.TextEncoder", "TextEncoder()")}}
返回一个新构造的TextEncoder，它将默认使用utf-8编码生成一个字节流。

方法

TextEncoder 接口不会继承任何方法。

{{domxref("TextEncoder.encode()")}}
返回包含使用此特定TextEncoder对象的方法编码的文本的Uint8Array。
