{{ApiRef("URL API")}} {{SeeCompatTable}}

URL 接口是一个包含若干静态方法的对象，用来创建 URLs。

当使用一个没有实现该构造器的用户代理时，可以通过 {{domxref("Window.URL")}} 属性来访问该对象（基于 Webkit 和 Blink 内核的浏览器均可用 Window.webkitURL 代替）。

{{AvailableInWorkers}}

属性

实现 {{domxref("URLUtils")}} 中定义的属性。

{{domxref("URLUtils.href")}}
包含完整 URL 的 {{domxref("DOMString")}}。
{{domxref("URLUtils.protocol")}}
包含 URL 协议名的 {{domxref("DOMString")}}，末尾带 ':'。
{{domxref("URLUtils.host")}}
包含 URL 域名，':'，和端口号的 {{domxref("DOMString")}}。
{{domxref("URLUtils.hostname")}}
包含 URL 域名的 {{domxref("DOMString")}}。
{{domxref("URLUtils.port")}}
包含 URL 端口号的 {{domxref("DOMString")}}。
{{domxref("URLUtils.pathname")}}
以 '/' 起头紧跟着 URL 文件路径的 {{domxref("DOMString")}}。
{{domxref("URLUtils.search")}}
以 '?' 起头紧跟着 URL 请求参数的 {{domxref("DOMString")}}。
{{domxref("URLUtils.hash")}}
以 '#' 起头紧跟着 URL 锚点标记的 {{domxref("DOMString")}}。
{{domxref("URLUtils.username")}}
包含在域名前面指定的用户名的 {{domxref("DOMString")}}。
{{domxref("URLUtils.password")}}
包含在域名前面指定的密码的 {{domxref("DOMString")}}。
{{domxref("URLUtils.origin")}} {{readonlyInline}}
返回一个包含协议名、域名和端口号的 {{domxref("DOMString")}}。
{{domxref("URLUtils.searchParams")}}
返回一个用来访问当前 URL GET 请求参数的 {{domxref("URLSearchParams")}} 对象。

构造器

{{domxref("URL.URL", "URL()")}}
使用给定参数创建并返回一个 URL 对象。

方法

URL 接口实现的在 {{domxref("URLUtils")}} 中定义的方法。

{{domxref("URLUtils.toString()")}}
返回一个包含完整 URL 的 {{domxref("DOMString")}}。它是 {{domxref("URLUtils.href")}} 的别名，但区别在于 toString 不能用于修改值。

静态方法

{{ domxref("URL.createObjectURL()") }}
返回一个{{domxref("DOMString")}} ，包含一个唯一的blob链接（该链接协议为以blob:，后跟唯一标识浏览器中的对象的掩码）。
{{ domxref("URL.revokeObjectURL()") }}
销毁之前使用{{ domxref("URL.createObjectURL()") }}方法创建的URL实例。

Chrome 浏览器无效

为了在chrome中使用, JSM and Bootstrap scope, 你必须像下面这样引入:

Cu.importGlobalProperties(['URL']);

URL可在Worker下使用。
