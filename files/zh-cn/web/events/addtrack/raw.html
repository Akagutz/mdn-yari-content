An addtrack  is fired when the browser adds a track to a {{domxref("MediaStream")}} (such as when a {{domxref("RTCPeerConnection")}} is renegotiated or a stream being captured using {{domxref("HTMLMediaElement.captureStream()")}} gets a new set of tracks because the media element being captured loaded a new source.

The addtrack event does not get fired when JavaScript code explicitly adds tracks to the stream (by calling {{domxref("MediaStream.addTrack", "addTrack()")}}).

An event handler for this event can be added by setting the {{domxref("MediaStream.onaddtrack")}} property, or by calling {{domxref("EventTarget.addEventListener", "addEventListener()")}} on the MediaStream which you want to monitor for track additions.
